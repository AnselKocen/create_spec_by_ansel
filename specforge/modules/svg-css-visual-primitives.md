# 高精度 SVG / CSS 可视化原语模块

当用户明确要求“高精度 SVG”“SVG 要求极高”“可视化是核心”“互动模拟”“参数模拟”“图表/地图/结构图/实验台/机制图必须精致”等能力时，最终 `spec.md` 不能只写“高精度”“分层”“酷炫”“SVG/Canvas 实现”。必须写到实现模型能照着绘制的颗粒度。

本模块用于把抽象视觉要求下沉为**绘制手册级指南**。物理、化学、地理、生物、医学、工程、地图、数据图表、机制模拟等产品都适用。

## 触发条件

只要满足任一条件，最终 spec 必须包含本模块要求的专项内容：

- 用户明确说 SVG 要求高、高精度 SVG、SVG/CSS 要好、可视化要强。
- 产品核心体验依赖互动模拟、参数探索、实验台、地图、机制图、结构图、图表、流程图、剖面图或网络图。
- spec 中声明核心可视化由 SVG / Canvas / CSS / HTML 实现，且不是普通装饰。
- 产品不允许用静态图片替代核心可视化。

## 最终 spec 必须写出的内容

在“视觉风格规范”或“核心机制约束”中，必须包含以下 6 类内容。章节标题可根据产品主题改名，但内容职责不能省略。

### 1. 可视化原语速查

必须列出本产品常见可视化对象，并说明：

| 字段 | 必须说明什么 |
|------|--------------|
| 要画的东西 | 具体对象，如等高线、细胞膜、烧杯、力箭头、食物网、气候曲线 |
| 推荐技术 | SVG / Canvas / HTML/CSS，或混合方案 |
| 为什么 | 为什么该技术适合这个对象 |
| 关键绘制点 | 形状、层级、标注、动态、数据编码要点 |

写法应接近：

```text
| 等高线 | SVG path + stroke-dasharray | 线条精确，方便按海拔分组和 hover 高亮 | 闭合线、标高标签、坡陡处线距更密 |
```

### 2. SVG / CSS 质感配方

必须写出本领域常见视觉对象的“怎么画出质感”，不能只写“有质感”。根据主题选择 6-12 类即可，不必覆盖宇宙所有对象。

每条至少包含：

- 形状基础：path、circle、polygon、rect、clipPath、mask、symbol/use。
- 填充方式：linearGradient、radialGradient、pattern、半透明 fill。
- 描边方式：stroke 宽度、linecap、linejoin、虚线、双层描边。
- 光影/质感：filter、feGaussianBlur、drop-shadow、高光条、内阴影、纹理。
- 动态方式：CSS transition、transform、stroke-dashoffset、opacity、path morph 或 Canvas 粒子。

示例风格：

```text
| 玻璃器皿 | 极低 alpha 填充 + 单侧白色高光条 + 半透明蓝灰描边；液体通过 clipPath 限制在器皿内部 |
| 能量/场线 | 多层 stroke 叠加，底层模糊辉光，前景细线；沿 path 移动小光点表达方向 |
```

### 3. 领域专属动效模式

必须写出哪些元素应该动、如何动、动效服务什么理解目标。

至少包含：

- 参数变化时如何插值，而不是瞬切。
- 哪些对象适合 CSS/SVG 动画。
- 哪些对象必须用 Canvas 或 `requestAnimationFrame`。
- 哪些动效是装饰，哪些动效是核心机制演示。
- `prefers-reduced-motion` 下保留哪些核心演示、关闭哪些装饰动效。

动效描述必须具体到对象，例如：

```text
降水粒子沿云带下落，密度映射降水强度；风向变化时粒子横向漂移方向随风向旋钮连续旋转。
```

### 4. 标注与数据编码原则

必须说明：

- 标注如何锚定：引线、小圆点锚点、贴边、跟随对象移动。
- 标注如何避让：多标注错开、动态避让、移动端折叠。
- 单位/数值如何显示：等宽字体、数值平滑变化、单位贴近数值。
- 图例如何解释颜色、线宽、透明度、符号、粒子密度、节点大小。
- 答题阶段哪些标注可以隐藏，哪些必须保留防止误判。

### 5. 交互态视觉暗示

必须为核心可交互对象写出具体状态，不得只写“有 hover/active”。

至少包含：

| 状态 | 应说明的内容 |
|------|--------------|
| hover | 轮廓、辉光、阴影、标签、cursor 怎么变 |
| dragging / adjusting | 对象如何跟随、是否放大、是否显示辅助线 |
| locked | 答题锁定态如何降噪，哪些控件禁用 |
| selected | 选择态如何保持可读，不破坏原始数据色 |
| correct / incorrect | 反馈色如何短时出现，如何回到原图层语义 |
| disabled | 不可操作态如何表示，不误导用户 |

### 6. 常见画错点 / 禁止降级

必须写本领域常见视觉错误，至少 5 条。它们要具体到画面和机制，不只是“不要错误”。

示例：

- 地理：河流不得逆坡流；等高线不得随意相交；气候图坐标轴不得省略；风场箭头不得穿过图例；热力图不能把温度、人口、风险全部用同一红色编码。
- 生物：细胞膜不能画成普通实心圆；细胞器不能无层级漂浮；DNA 螺旋不能只做装饰而不参与机制；血氧颜色不能和错误反馈混淆；遗传概率不能只靠随机动画表现。
- 化学：玻璃器皿不能是不透明实心块；沉淀颜色必须符合事实；气泡不能静止贴在液体中；化学式上下标必须正确。
- 物理：力箭头方向不能反；场线不能交叉；矢量长度不能与数值无关；轨迹不能违反公式。

## 写作深度要求

最终 spec 中的高精度 SVG/CSS 指南至少应达到以下深度：

- 不是一句“使用高精度 SVG”。
- 不是只列对象清单。
- 必须能指导实现者选择 SVG / Canvas / CSS / HTML。
- 必须说明 6 类以上常见视觉对象的质感做法。
- 必须说明交互态和反馈态如何画。
- 必须说明常见画错点。
- 必须把这些要求写入验收标准。

## 与 Canvas 的边界

SVG 适合：

- 精确结构、标注、图层、可点击热区、图例、坐标轴、矢量路径、器材骨架。

Canvas 适合：

- 大量粒子、连续流场、扩散、碰撞、实时轨迹、云雾、火焰、液体、生态群体运动。

HTML/CSS 适合：

- 参数面板、数值读数、按钮、图例、卡片、toast、表格、布局、状态反馈。

同一个可视化可以混用三者。最终 spec 应明确“骨架用 SVG、连续动态用 Canvas、控制/读数用 HTML/CSS”这类分工。

## 验收标准必须增加

凡触发本模块，最终 spec 的验收标准必须包含：

- [ ] 核心可视化不是静态图片、不是纯文字、不是无交互 SVG 装饰。
- [ ] 已按产品领域写出可视化原语速查。
- [ ] 已按产品领域写出 SVG/CSS 质感配方。
- [ ] 关键对象有标注、图例、单位或状态说明。
- [ ] hover、dragging、locked、selected、correct、incorrect 等状态有视觉反馈。
- [ ] 参数变化有连续过渡或实时动画，不能瞬切。
- [ ] 常见画错点已避免。
- [ ] `prefers-reduced-motion` 下仍保留核心机制演示。

## 访谈阶段应追问

如果用户提出高精度 SVG 或互动模拟，但没有说明重点，访谈阶段应补问：

- 哪些对象必须用 SVG 精细绘制？
- 哪些过程需要 Canvas 粒子或连续动画？
- 是否更重视结构准确、质感表现、交互反馈、还是数据判读？
- 有没有参考物理/化学那样的绘制强度要求？

如果用户已经明确“像物理/化学那样细”，不必继续追问，直接在 `spec.gameplay.md` 和最终 `spec.md` 中加入本模块要求。
