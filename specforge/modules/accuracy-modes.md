# 内容准确性模式

工具必须询问用户选择准确性模式，并将结果写入最终 spec。

## 严谨模式

适合：物理、化学、医学、法律、历史、真实传记、教材类学习产品。

要求：
- 不允许虚构关键知识。
- 公式、单位、方程式、时间线、人物关系、实验现象必须正确。
- 题目答案必须经过验证。
- 错误选项应对应真实常见错误，而不是随机干扰项。
- 验收标准必须包含“内容准确性验收 / 自检”。

## 半严谨模式

适合：科普、历史改编、人物体验、轻学习游戏。

要求：
- 核心事实和关键概念必须正确。
- 允许叙事化、戏剧化表达，但不能改变关键因果和知识结论。
- 验收标准应检查核心事实、术语、结构是否合理。

## 幻想模式

适合：恐怖、奇幻、人格测试、抽象游戏、非真实世界设定。

要求：
- 允许自由发挥和原创设定。
- 内容必须和随附内容文件 / `skills` 有内在联系，不能只是套皮。
- 重点验收玩法完整度、氛围、视觉、文本质量和一致性。
- 如果包含真实知识点，仍需避免明显错误。
