### ⚠️ 核心设计理念

> **知识点、答题、动画是同一个东西的三个面，不是三个独立模块的拼接。**
>
> 每一个知识点对应一个可视化实验场景，这个场景贯穿「讲解→探索→答题→反馈演示」四个阶段。学生始终在同一个实验台上学习和作答——讲解时实验演示现象，探索时实验开放操作，答题时实验锁定状态等待作答，反馈时实验直接演示答案。
>
> **如果把实验动画去掉只剩文字和选择题，这个产品就没有存在的价值。**
>
> 技术选型：
> - **SVG**：实验器材（烧杯、试管、锥形瓶、酒精灯…）、分子结构图、化学方程式图示——矢量精确，支持 CSS 动画和标注，天然适合表达玻璃器皿的透明质感
> - **Canvas + JS**：粒子运动模拟（分子热运动、布朗运动）、实时反应动画（气泡生成、沉淀下落、颜色渐变）、火焰效果——逐帧渲染，支持大量动态元素和实时参数响应
> - **CSS 动画**：UI 过渡效果、液体晃动、发光/闪烁、背景氛围——提升界面精致感
>
> **禁止降级**：不可以用静态图片替代实验动画，不可以用纯文字描述替代可视化，不可以跳过探索阶段的交互功能。

## 四、视觉风格规范

> 本节强制约束游戏的视觉表现。化学游戏的视觉目标是「现代化学实验室」——干净、明亮、精致，器皿有玻璃质感，液体有真实色彩，反应有生动现象。

### 整体气质

**现代实验室感**：学生打开页面应该感觉站在一间设备精良的化学实验室里，面前是干净的实验台、透亮的玻璃器皿、色彩鲜艳的试剂。不是课本插图，是「能动手」的虚拟实验台。

### 背景系统

> 背景不是装饰，是氛围的基础。本项目使用全局固定背景图，所有 UI 和可视化浮在这张图之上。背景图必须由开发者在构建时编写提示词并调用生图模型生成。**禁止用 CSS 渐变、纯色或 SVG 冒充背景图。**

**背景图生成要求**：
- 提示词方向：**现代化学实验室背景**——干净整洁的实验台、玻璃器皿、试剂架，明亮的实验室灯光，整体色调清亮干净，无主体人物，以环境/场景为主
- 参考关键词：`modern chemistry laboratory background, clean bench, glass equipment, bright lighting, professional, no people, cinematic atmosphere`
- 如果无法调用生图模型，**必须停下来告知用户，不可用 CSS 替代**

**什么是"生图"**：
- ✅ 调用生图模型，输入提示词，获得一张真实的图像文件
- ❌ 用 CSS 渐变、box-shadow 画出来的图形 — 这不是生图
- ❌ 用 SVG path 手动描绘 — 这不是生图
- ❌ 用 Canvas 代码绘制 — 这不是生图

**实现要求**：
- `background-image` 固定铺满视口，`background-size: cover; background-position: center; background-attachment: fixed`
- 背景图上方叠加一层半透明中性灰遮罩，防止背景图案干扰前景内容的可读性。遮罩必须使用中性灰（`rgba(0,0,0,0.30)` 左右），不可使用带色调的色值，防止遮罩对前景文字色温产生干扰。化学版背景明亮，遮罩透明度略低于物理版，可根据页面内容密度微调

> 原则：背景图是固定的氛围底图，所有动态效果和注意力焦点在前景的器皿、反应和 UI 上。背景不应分散对核心内容的注意力。

### 首页图层层级原则

> 背景图和遮罩必须永远位于所有 UI 内容之下，不得遮挡标题、按钮、实验台等任何前景元素。

- 推荐用 CSS `background-image` 属性实现背景图（作用于 body 或最外层容器），天然不会覆盖任何内容
- 如果用独立 div 实现背景图，必须确保该 div 不遮挡 UI——开发者自行选择最稳妥的实现方式
- 遮罩层同理，必须位于所有 UI 元素之下，背景图之上
- 粒子 Canvas 不拦截 UI 点击事件

### 标题页氛围粒子（必须实现）

> 标题页必须有与实验室主题匹配的氛围粒子效果，强化沉浸感。

- **粒子类型**：漂浮的小气泡 或 飘散的分子符号小点（颜色取主题色系的半透明亮色）
- 数量 15-25 个，随机分布在背景层，缓慢上浮或漂移
- 粒子在标题页全程可见；进入章节后可保留低密度版本作为氛围层，或关闭（不强制）
- 用 Canvas 或 CSS animation 实现均可，不影响性能
- 尊重 `prefers-reduced-motion`（关闭粒子运动，保留静态粒子）

### UI 元素风格

> 所有 UI 元素走「干净实验室」路线——清晰、圆润、有质感，但不浮夸。

**卡片与面板**：
- 白色或极浅色底 + 轻微阴影（`box-shadow`）+ `backdrop-filter: blur()` 毛玻璃效果（可选）
- 边框用浅色细线或无边框，依靠阴影层次区分
- 圆角稍圆润（8-12px），比物理版更柔和

**按钮**：
- 主按钮：实色填充（从强调色取）+ 白色文字 + 轻微阴影
- 次按钮：透明底 + 强调色描边 + 强调色文字
- hover：轻微上浮（translateY -1px）+ 阴影加深
- active：按下（translateY +1px）+ 阴影减弱
- 禁用态：灰色底 + 降低 opacity

**滑块与控件**：
- 滑轨用圆角细条，填充渐变色（如温度从蓝到红）
- 滑块手柄用圆形白色按钮 + 阴影，拖拽时手柄放大 + 阴影加深
- 数值显示用等宽字体，数值变化时有平滑过渡动画（不跳变）

**章节过渡**：
- 实验室风格的过渡：试管液体注入、烧瓶冒泡、分子组装、元素周期表元素飞入
- 时长 1.5-3s，不能瞬切，也不能太慢拖沓

### SVG/Canvas 可视化风格

> 可视化是产品核心——模拟真实实验。器皿要有玻璃感，液体要有颜色和流动感，反应要有可见的现象变化。

**线条**：
- 器皿轮廓用清晰利落的线条，stroke-width 1-2px
- 描边颜色用半透明的蓝灰色（不是纯黑），模拟玻璃边缘
- 辅助线（标注线、水位线）用低透明度虚线

**填充**：
- 器皿本体用极低 alpha 的浅蓝色填充，模拟玻璃的微弱底色
- 液体用半透明的鲜艳色（每种试剂有自己的颜色，alpha 0.5-0.8）
- 气体/烟雾用极淡的渐变色块 + 模糊

**实验器材绘制**：
- 所有玻璃器皿必须有透明质感——不能画成不透明的实心色块
- 器皿内的液体要能透过玻璃壁看到
- 金属部件（铁架台、夹子、瓶盖）用灰色渐变模拟金属光泽

**标注**：
- 深色文字（深灰或深蓝灰）+ 细引线 + 小圆点锚点
- 标注跟随实验变化时平滑移动，不跳变
- 多标注不重叠，有避让逻辑
- 化学式用正确的上下标格式（如 H₂O、CO₂）
- 数值标注用等宽字体，单位跟在数值后

**动画气质**：
- **实验模拟**——液体倒入时有流动感、加热时有气泡上升、沉淀缓慢下落、颜色渐变平滑过渡。强调「这是活的实验」
- 参数变化时动画连续插值过渡，**禁止重绘整个画面**
- 动画帧率 ≥ 30fps，Canvas 动画使用 `requestAnimationFrame`

### 化学可视化原语速查

> 本节定义化学实验场景中常见图形元素的**绘制工具和质感方向**。开发者根据知识点内容自行组合——这是风格指南，不是清单。

#### 选什么工具画

| 要画的东西 | 用什么 | 为什么 |
|-----------|--------|--------|
| 玻璃器皿（烧杯/试管/锥形瓶/量筒/蒸馏瓶…） | SVG `<path>` + 渐变/滤镜 | 形状精确，天然支持透明度和高光，方便标注刻度 |
| 液体（静态液面 + 颜色） | SVG `<path>` + 半透明填充 | 形状与器皿吻合，颜色变化用 CSS transition |
| 气泡上升、沉淀下落 | Canvas 粒子系统 | 数量多、每帧更新位置和大小 |
| 火焰（酒精灯/本生灯） | Canvas 粒子 或 SVG 渐变动画 | Canvas 更自然，SVG 更简单——视复杂度选择 |
| 分子/原子模型（球棍模型） | SVG 圆 + 线 | 结构精确，方便标注元素符号和键类型 |
| 化学反应动画（颜色变化/气体释放/沉淀生成） | Canvas + JS | 需要逐帧控制多种同时发生的现象 |
| 实验装置支架（铁架台/夹子/石棉网） | SVG | 静态结构，需精确尺寸 |
| 数值面板（温度/pH/浓度显示） | HTML + CSS | DOM 元素实时更新最简单 |

> **可以混用**：同一个实验里 SVG 画器皿骨架 + Canvas 画气泡和火焰 + HTML 做数值面板，完全合理。

#### 怎么让它有质感

| 质感目标 | 一句话做法 |
|---------|-----------|
| 玻璃器皿 | `rgba` 极低 alpha 填充（0.05-0.15）+ 单侧白色高光条（器皿左侧 20-30% 位置，opacity 0.25-0.4）+ 半透明蓝灰描边。液体要能透过杯壁看到 |
| 液体 | 半透明鲜艳填充（alpha 0.5-0.8）+ 顶部弯月面弧线（两端比中间高 2-3px）+ 液面反光（顶部一条极细白色亮线） |
| 气泡 | 小圆 + 白色高光点偏左上 + 极淡描边（不是实心圆），从底部随机位置生成，上升时左右微摆 + 到液面破裂消失 |
| 沉淀 | 不规则小颗粒（小圆或小多边形），从液面缓慢下沉到容器底部堆积，颜色不透明 |
| 火焰 | 底部蓝色 → 中段橙黄 → 顶部透明的渐变，外轮廓有轻微随机晃动（每帧微调控制点） |
| 金属器材 | 线性渐变（左侧偏亮 → 右侧偏暗）模拟圆柱金属光泽 + 细反光线 |
| 烟/气体 | 极淡的渐变色块（opacity 0.1-0.3）+ 向上缓慢飘动 + 逐渐扩散变淡消失 |
| 粉末/固体 | 小圆点群密集排列 + 颜色不透明 + 可有微弱纹理变化 |

#### 颜色怎么分配

> 化学中颜色承载物质信息——颜色选择必须尽量接近真实试剂。

- 液体颜色应尽量接近真实试剂颜色（CuSO₄ 溶液=蓝色，FeCl₃=黄棕色，KMnO₄=紫色，NaOH=无色透明，石蕊=紫→红/蓝）
- 无色液体（水、稀酸、稀碱）用极淡的浅蓝色表示（纯透明在屏幕上看不见），标注「无色」
- 气体用对应颜色的极淡版本（Cl₂=浅黄绿，NO₂=浅棕红，无色气体用微弱的折射线条暗示）
- 沉淀颜色必须准确（Cu(OH)₂=蓝色，Fe(OH)₃=红棕色，AgCl=白色，BaSO₄=白色）
- 指示剂颜色变化必须准确（酚酞：酸性无色/碱性粉红；石蕊：酸性红/碱性蓝）
- 同一画面中不同物质用颜色区分，保证辨识度

#### 标注原则

- 器皿旁标注名称（如「锥形瓶」「量筒」），液体旁标注化学式或名称
- 刻度线和刻度数值直接画在器皿壁上（模拟真实刻度）
- 温度、pH 等读数用数值面板显示，变化时平滑过渡
- 化学方程式标注用正确的上下标和配平系数
- 答题阶段可选择性隐藏部分标注作为题目条件

#### 交互态视觉暗示

- 可拖拽的器皿/试剂瓶：加轻微上浮阴影 + hover 时边框高亮 + cursor: grab
- 可调节的控件（酒精灯火焰大小、滴管滴速）：hover 时控件高亮
- 答题锁定态：器皿不可拖拽 + 降 opacity + cursor: default
- 拖拽中：器皿跟随鼠标 + 阴影加深加大
- 释放/倒入：液体流动动画 + 目标器皿接收

### 配色方案

| 用途 | 方向 | 参考值（不强制） | 说明 |
|------|------|-----------------|------|
| 60% 主色（背景/表面） | 明亮实验室色 | 如 #f5f7fa / #f0ece4 | 浅灰蓝 / 浅米白 / 浅青灰，干净明亮 |
| 30% 辅助色（文字/边框） | 深色系 | 如 #2d3748 / #4a5568 | 深灰 / 深蓝灰，保证可读性 |
| 10% 强调色（按钮/高亮） | 活力色 | 如 #3b82f6 / #10b981 | 亮蓝 / 翠绿 / 琥珀色，用于交互元素 |
| 成功色 | 翠绿系 | 如 #10b981 | 答对反馈、正确操作高亮 |
| 错误色 | 暖红系 | 如 #ef4444 | 答错反馈，不刺眼 |
| 试剂色板 | 真实试剂色 | — | 每种化学物质用接近真实的颜色，详见「颜色怎么分配」 |

> 配色原则：整体明亮干净。试剂颜色是画面中最鲜艳的部分，自然成为视觉焦点。文字与背景对比度 ≥ 4.5:1。具体色值由开发者在以上方向内自行决定。

### 必须实现的视觉反馈

| 用户操作 | 视觉反馈要求 |
|---------|------------|
| 进入章节 | 章节过渡动画（试管注液/分子组装/元素飞入），不能瞬切 |

| 完成章节 | 知识卡片总结 + 成就展示 |
| 屏幕切换 | 淡入淡出过渡，不能瞬切 |
| 悬停可交互元素 | 高亮/阴影加深/微上浮，必须有明确反馈 |

### 动效标准

- **标题页进场**：背景图 → 主标题淡入上浮 → 副标题淡入 → 按钮淡入，依次触发，总时长 ≤ 1.5s
- **标题页按钮**：常态有轻微呼吸脉动（scale 或 box-shadow，周期 3s），hover 轻微上浮 + 阴影加深，click 短暂按压（scale 0.97）
- **标题页 → 章节主页**：淡出淡入，背景图保持不动，不瞬切（500ms 左右）
- 所有 UI 过渡时长 200-800ms
- 使用 ease 或 cubic-bezier 缓动曲线，禁止线性运动
- 液体流动/倒入动画 500-1500ms（视液量而定）
- 颜色渐变反应 800-2000ms（模拟真实反应速度，不能瞬变）
- 气泡上升速度自然（不能太快也不能太慢），有随机性
- 尊重用户的 `prefers-reduced-motion` 系统偏好（简化气泡和流动动画，保留核心反应动画）
- 可视化动画应能暂停/重播
- 动效必须有目的（辅助理解 / 提供反馈 / 模拟真实现象），禁止无意义的闪烁

## 七、信息结构

### 导航结构

```
标题页
├── 开始实验 → 章节主页
└── 继续学习 → 章节主页（恢复存档进度，仅有存档时显示）

章节主页（核心导航枢纽，每次完成章节后返回这里）
├── 章节列表（纵向或横向排列）
│   ├── 已完成章节 → 亮色 + 完成标记 + 可点击回看
│   ├── 当前可进入章节 → 高亮 + 引导动效 + 可点击进入
│   └── 未解锁章节 → 灰色/锁定 + 不可点击
└── 整体进度展示（如 2/6 章已完成）

学习中（线性推进，章内不可跳转）
├── 章节过渡屏
├── 知识点闭环（重复 3-4 次/章）
│   ├── 讲解阶段（实验演示 + 文字）
│   ├── 探索阶段（交互式实验）
│   ├── 答题阶段（3 选 1）
│   └── 反馈演示（实验动画演示答案）
└── 章末总结（知识卡片 + 成就展示）→ 返回章节主页

结局总结页（全部章节完成后触发）
├── 成就墙（所有已解锁成就）
├── 各章回顾
└── 返回章节主页
```

> 本游戏为线性解锁路径——必须通过第 N 章才能解锁第 N+1 章。但章节主页允许回看已完成章节。

### 游戏界面布局

| 界面 | 核心内容 | 交互方式 |
|------|---------|---------|
| 标题页 | 游戏名称（主题化字体特效）+ 副标题 + AI 生成实验室背景图（灰度遮罩）+ 开始/继续按钮 + 氛围粒子 | 点击按钮 |
| 章节主页 | 章节列表（已完成/当前/未解锁状态区分）+ 整体进度 | 点击章节进入 |
| 章节过渡 | 章标题 + 本章核心概念预告 + 过渡动画 | 点击"进入实验室" |
| 讲解阶段 | 上方：SVG/Canvas 实验台可视化（动画播放）；下方：讲解文字（逐段显现）+ 继续按钮 | 点击继续 |
| 探索阶段 | 中央：交互式实验台（带拖拽器皿/滑块）；下方：参数面板 + 引导文字 + "来挑战"按钮 | 拖拽/滑动 + 点击 |
| 答题阶段 | 上方：实验台（锁定特定状态）；下方：题目文字 + 3 个选项按钮 | 点击选项 |
| 反馈演示 | 上方：实验台（播放答案演示）；下方：反馈文字 + 继续/重试按钮 | 点击继续或重试 |
| 章末总结 | 方程式卡片 + 要点列表 + 本章成就展示 | 点击继续 |
| 结局总结 | 成就墙（卡片网格）+ 各章一句话回顾 + 重新开始按钮 | 点击"重新开始" |

### 标题页设计规范

**字体层级**：
- 主标题字号 ≥ 40px（移动端）/ ≥ 56px（桌面端），有主题化字体特效（实验室风格：干净有力的字体 + 轻微投影或亮色填充）
- 副标题字号约为主标题的 40-50%，字重轻，颜色比主标题浅，体现层次
- 按钮文字字间距略宽，简洁有力

**进场动画**（页面加载时依次触发）：
1. 背景图淡入（300ms）
2. 粒子开始漂浮
3. 主标题上浮淡入（500ms，delay 200ms）
4. 副标题淡入（400ms，delay 600ms）
5. 按钮上浮淡入（400ms，delay 900ms）

**按钮动效**：
- 常态：轻微呼吸脉动（scale 1 → 1.03 → 1，周期 3s）
- hover：轻微上浮（translateY -2px）+ 阴影加深
- click：短暂按压（scale 0.97 + translateY +1px）

**标题页 → 章节主页过渡**：点击按钮后，标题页内容淡出，章节主页淡入（500ms），背景图保持不动

## 九、交互设计

### 交互模式选择

**本项目选择**：新手友好 + 实验操作式

**选择原因**：目标用户是学生，基本操作（点击推进 + 选择答案）要极简，但探索阶段需要拖拽器皿、调节参数等操作来模拟真实实验体验。

### 操作映射

| 操作 | 实现方案 | 说明 |
|------|---------|------|
| 推进讲解 | 点击"继续"按钮 | 文字和实验动画同步，按钮在动画播完后出现 |
| 交互探索 | 拖拽器皿/试剂瓶/调节滑块 | 实验实时响应，数值面板同步更新 |
| 回答问题 | 点击 3 个选项之一 | 选后进入反馈演示，答错可重试 |
| 观看反馈 | 自动播放 + 点击继续 | 反馈动画播完后出现继续按钮 |
| 存档 | 自动 | 每次状态变化自动写入 localStorage |
| 读档 | 点击标题页"继续学习" | 进入章节主页，恢复存档进度 |

### 交互探索的具体模式

> 探索阶段的交互方式取决于化学内容，以下为常见模式：

| 交互模式 | 适用场景 | 操作方式 |
|---------|---------|---------|
| 拖拽倒入 | 将试剂从一个容器倒入另一个 | 拖拽试剂瓶到目标容器上方，松开触发倒入动画 |
| 滴加控制 | 滴管/滴定管逐滴加入 | 点击滴管或拖拽滴速滑块，观察每滴引起的变化 |
| 温度调节 | 加热/冷却实验 | 拖拽温度滑块或调节酒精灯火焰大小 |
| 浓度调节 | 配制不同浓度溶液 | 拖拽滑块改变溶质量/溶剂量 |
| 开关切换 | 显示/隐藏分子模型视图、微观/宏观切换 | 点击 toggle 按钮 |
| 播放/暂停 | 反应过程模拟 | 点击播放按钮，可暂停在任意时刻观察 |

### 交互反馈

| 操作 | 视觉反馈 | 状态变化 |
|------|---------|---------|
| 阅读讲解 | 文字显现 + 实验动画播放 | 进入探索阶段 |
| 拖拽倒入试剂 | 液体流动动画 + 目标容器液面上升 + 可能的颜色变化/气泡/沉淀 | 无状态变化 |
| 调节参数 | 实验实时变化 + 数值更新 | 无状态变化 |
| 答对 | 实验演示正确结果 + 成功动效 + 鼓励文字 | 可能触发成就 |
| 答错 | 实验演示正确过程 + 温和提示 + 重试按钮 | 记录错误，不扣分 |
| 探索中特殊操作 | 可能触发创意成就的 toast | 解锁隐藏成就 |
| 进入新章 | 章节过渡动画 | currentChapter +1 |
| 通关 | 跳转结局成就墙 | finished = true |

## 十、核心机制约束

> 以下是**不可更改的规则**，开发时必须遵守。

### 科学准确性（极其重要）

- 化学方程式必须正确配平——**这是本产品的底线**
- 反应现象（颜色变化、气体生成、沉淀生成、温度变化）必须与真实实验一致
- 物质的颜色、状态、溶解性必须正确（不能把白色沉淀画成蓝色）
- 探索阶段的参数范围必须化学上合理（如 pH 0-14，温度不能低于绝对零度，浓度 ≥ 0）
- 解析必须严谨，不能有模糊或误导性表述
- 题目的正确答案必须经过验证

### 知识自检规则

> **AI 在生成内容时，必须对所有涉及的化学知识执行自检。** 这是强制流程，不可跳过。

自检清单（每个章节输出后逐条核对）：
1. **方程式验证**：本章出现的所有化学方程式是否正确配平？反应条件（加热、催化剂、点燃等）是否标注？
2. **现象验证**：讲解和可视化中描述的反应现象是否与真实实验一致？（颜色、状态、气味等）
3. **物质属性验证**：涉及的物质颜色、溶解性、酸碱性是否正确？
4. **数值验证**：题目中的计算过程是否正确？答案是否经过反算验证？
5. **可视化一致性**：可视化中的反应现象是否与化学原理一致？（如加入酸后石蕊确实变红）
6. **选项设计**：错误选项是否对应真实的学生常见错误（不是随机干扰项）？正确选项的解析是否充分？

> 如果自检发现错误，必须修正后再输出。在输出末尾附上自检结果摘要。

### 知识点四阶段闭环规则

> **这是本游戏区别于普通题库的核心机制。** 每个知识点必须走完四个阶段，且四个阶段共用同一个实验场景。

**阶段 1：讲解**
- 一边说原理一边播实验动画，学生看到化学现象
- 可以分多段讲解（1-3 段），每段 60-120 字，每段对应不同的 vis_state（如：先演示实验操作→再展示现象→最后引出方程式）
- 通俗易懂，先给现象再给原理
- 关键术语首次出现时加粗并解释
- 动画和文字同步——文字说到「加入盐酸」，动画就演示倒入的过程

**阶段 2：探索**
- 开放实验操作，学生自己拖拽/调节
- 提供引导文字（如"试试改变溶液的 pH，观察指示剂颜色变化"）
- 探索没有对错，纯粹建立直觉
- 实验实时响应操作，数值面板同步更新
- 某些特殊操作（如把浓度拉到极端值、混合不常见的试剂组合）可触发创意成就

**阶段 3：答题**
- **题目必须基于前两个阶段的实验场景出题**——考察学生对实验现象的理解
- 实验锁定在特定状态（如锁定某试剂、隐藏某现象）
- 恰好 3 个选项，正确答案只有 1 个
- 选项文字清晰无歧义

**阶段 4：反馈演示**
- **不管答对还是答错，都用实验动画演示**
- 答对：实验播放正确结果 + 高亮关键变化 + 肯定 + 为什么对（30-80 字）
- 答错：实验先演示「你选的会怎样」再演示「正确的是怎样」+ 不批评 + 指出错在哪（50-100 字）
- 答错可以重试（但不重复触发成就）
- 语气鼓励——"没关系，我们再看看实验结果" 而不是 "错了"

### 四阶段的可视化一致性

- **禁止**每个阶段造一个不相关的实验——四个阶段是同一个实验台的不同状态
- 讲解时：实验处于「演示模式」，按预设流程播放
- 探索时：实验处于「交互模式」，操作参数开放
- 答题时：实验处于「锁定模式」，特定操作和参数固定
- 反馈时：实验处于「演示模式」，播放答案对应的过程

### 成就系统规则

> **本游戏不使用分数制。** 用成就替代评分，每章 3 个成就，成就要有个性和记忆点。

**每章 3 个成就**：

| 类型 | 触发条件 | 风格要求 |
|------|---------|---------|
| 全对成就 | 本章所有题首次答对 | 正面肯定，专家/大师风格。如"配平大师"、"酸碱平衡专家" |
| 全错成就 | 本章所有题首次答错 | 调皮调侃，不伤人。如"试管们在流泪"、"元素周期表震惊了" |
| 创意成就 | 探索阶段的特殊操作 | 彩蛋风格，奖励好奇心。如"把 pH 拉到 0 的勇者"、"发现了彩虹反应" |

成就规则：
- 成就名称 6 字以内，描述 30-50 字
- 解锁时弹出 toast 通知（从顶部滑入，3s 后自动消失）
- 结局页展示成就墙：已解锁的成就亮色显示，未解锁的灰色占位（可显示提示）
- 重试不重复触发成就
- 全对/全错判定基于**每道题的首次作答**——答错后重试答对不影响"首次答错"的记录
- 触发时机：章内**最后一道题首次作答完成后**立即判断，若条件满足则弹出 toast；不在每道题后单独判断
- 创意成就的触发条件由开发者根据实验内容设计，鼓励学生大胆探索
- "重新开始"会清除所有进度和成就，从头来过

### 章末知识总结

- 每章末尾 1 个总结场景
- 核心方程式大字居中展示（正确配平 + 反应条件）；本章有多条核心方程式时逐条展示，数据来自 `summary.formulas` 数组
- 2-3 个要点（bullet points）
- 一句话点睛（让学生记住的核心直觉）
- 展示本章已解锁的成就

### 结局总结

### 化学实验场景速查

> 以下列出常见化学实验可视化场景的要点，帮助开发者快速定位。

**酸碱反应**：
- 指示剂颜色变化——石蕊/酚酞在不同 pH 下的颜色，必须准确
- 中和反应——混合后温度升高（温度计读数变化），滴定曲线
- 酸 + 金属——气泡（H₂）生成 + 金属逐渐溶解 + 溶液颜色变化
- ⚠️ 常见错误：指示剂颜色画反、弱酸当强酸处理

**氧化还原**：
- 金属置换——如铁钉放入硫酸铜溶液，铁钉表面出现红色铜 + 溶液从蓝变浅
- 燃烧反应——火焰颜色（镁=白光，钠=黄，铜=绿）+ 生成物
- 电解——阳极/阴极产物不同，气泡分别在两极生成
- ⚠️ 常见错误：氧化剂和还原剂搞混、电极产物位置反

**溶液与沉淀**：
- 溶解过程——固体逐渐缩小消失 + 溶液逐渐变色
- 沉淀生成——两种溶液混合后沉淀颗粒从液面下落堆积
- 过滤——液体通过滤纸，滤液清澈，滤渣留在滤纸上
- ⚠️ 常见错误：沉淀颜色不准确、溶解度概念可视化不到位

**气体制备与收集**：
- 制取装置——锥形瓶+分液漏斗+导管+集气瓶
- 排水集气法/向上排空气法——气泡通过水面、气体充满集气瓶
- ⚠️ 常见错误：收集方法与气体密度不匹配

**有机化学（如涉及）**：
- 分子模型——球棍模型显示碳骨架和官能团
- 取代/加成反应——键的断裂和形成动画
- ⚠️ 常见错误：碳的四价不满足、手性分子画成对称

**热化学/能量**：
- 放热反应——温度计上升 + 容器外壁「发热」效果（微弱红色光晕）
- 吸热反应——温度计下降 + 容器外壁「变冷」效果（微弱蓝色光晕）
- 能量图（反应物→活化能→生成物能量曲线）
- ⚠️ 常见错误：放热/吸热反应的能量变化方向画反
