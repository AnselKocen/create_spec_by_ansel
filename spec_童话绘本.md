# 儿童童话绘本生成器设计文档

---

## 一、产品概览

| 维度 | 值 |
|------|-----|
| 产品名称 | 童话小书匠 |
| 输入内容来源 | 随本 spec 一起提供的完整内容文件 / `skills/`：书籍、文章、课程、人物、产品说明、故事梗概、抽象主题等 |
| 产品类型 | 儿童童话绘本生成器 / 儿童安全改编器 |
| 目标读者 | 3-10 岁儿童，默认 6-8 岁 |
| 默认安全目标 | `KIDS-6` 内部安全目标，向下兼容儿童友好内容 |
| 绘本长度 | 封面 + 8-12 个内页 + 结尾页 |
| 交互方式 | 鼠标点击 / 触屏点击 / 简单拖拽 / 温和选择 |
| 生图模式 | 必须真实 AI 生图：封面 + 每页插图 |
| 随附内容文件 | 完整内容文件会与本 spec 一起提供，通常位于 `skills/` 目录 |
| 准确性模式 | `fantasy + strict child-safety`，即幻想改写 + 严格儿童安全 |
| 核心主题 | 把复杂、成人化或抽象材料转成适合孩子阅读的童话绘本 |
| 目标体验 | 温柔、安全、有想象力、有道理，像一本可以互动的儿童绘本 |

### 核心设计理念

> **这是一个儿童绘本改编器，不是摘要工具、知识问答工具或原文忠实改写器。**
>
> 输入材料可以很复杂、成人化、恐怖、商业、学术或抽象，但输出必须始终变成适合儿童观看和阅读的童话绘本。产品不追求“事实是否完整还原”，而追求“孩子能安全地读、愿意读、读完获得一个温和正向的道理”。

最高优先级：

1. **儿童安全高于一切**：儿童安全高于素材还原、高于剧情刺激、高于事实准确。
2. **可以大幅童话化**：原文中的人物、机构、冲突、概念都可以被改成小动物、精灵、森林、星星、魔法道具或虚构王国。
3. **绝对不能教坏孩子**：不能把欺骗、霸凌、危险模仿、歧视、报复、绝对服从、过度功利包装成正向结果。
4. **不适合儿童就改写或阻断**：如果素材无法安全改写为儿童绘本，必须停止并给出 `BLOCKED`，不能硬编。

本产品的核心价值不是“讲对原材料”，而是“从任何材料中提取一个儿童可理解的温和意义，并变成一本安全、漂亮、可共读的童话绘本”。

### 实现范围说明

本 spec 面向下一步大模型或编码助手执行。可支持两种实现方式：

| 模式 | 说明 | 要求 |
|------|------|------|
| 构建时改编模式 | 实现者读取与本 spec 一起提供的完整内容文件 / `skills/`，在开发阶段生成一册固定绘本 | 适合单个 HTML 交付，所有故事和图片构建时写入数据 |
| 运行时生成模式 | 用户在页面输入材料，系统运行时生成绘本 | 必须接入文本生成模型和生图工具，不能只用本地 JS 假装生成 |

如果最终交付为单个 HTML 文件，默认采用“构建时改编模式”。如果要做成运行时生成工具，必须明确配置文本模型和生图 API。

### 开始前家长预览警告（固定模块）

最终产品必须在进入封面或第一页之前显示一个可爱但明确的家长预览警告。它是 P0 功能，不是可选说明页。

警告文案必须包含以下含义：

> 给大人的小提示：本绘本由 AI 生成，故事、画面和互动可能存在一定不确定性。请家长先完整试玩/阅读，确认适合孩子后，再给孩子观看。

实现要求：

- 警告页视觉可以可爱、温柔、像贴在绘本封面前的小纸条，但信息必须清楚。
- 警告页必须在首次进入绘本前出现，不得直接进入儿童阅读页。
- 主按钮文字建议为“我会先试玩”，不要写成“开始给孩子看”。
- 家长确认后进入封面页或家长预览模式。
- 该模块不能替代儿童安全闸门；它只是额外提醒 AI 生成内容存在不确定性，最终仍需家长试玩确认。

## 二、涉及的 SKILL / 内容文件清单

> **完整内容文件会与本 spec 一起提供，通常位于项目根目录的 `skills/` 文件夹中。构建产品时，必须读取每个文件的完整内容，并从中提取本 spec 需要的内容。**

这些内容文件是本绘本生成器的核心输入源，不是可选参考资料。实现者必须从中提取可被儿童安全改编的主题、角色原型、事件关系、知识概念、情绪线索、风险内容和正向寓意，再按本 spec 的儿童安全规则转译成绘本数据。

| 内容文件路径 | 覆盖模块 | 在本产品中的用途 |
|-------------|----------|------------------|
| `{{CONTENT_FILE_PATH}}` | `{{COVERED_MODULE}}` | 提取可儿童化改编的核心意义、角色/概念原型、风险内容、故事主题、画面意象和结尾道理 |

必须遵守：

- 必须读取每个内容文件的完整正文，不能只看索引摘要。
- 不能只读文件名、标题或目录后自行编造绘本核心内容。
- 不能把内容文件原样复述给儿童，必须经过儿童安全闸门、风险删改和童话转译。
- 内容文件中不适合儿童的部分必须进入 `safety_report` 和 `source_material.discarded_or_rewritten_elements`，说明删改、弱化、隐喻化或阻断原因。
- 如果内容文件为空、缺失或无法读取，必须停下来告知用户，不能凭标题硬编一册绘本。

## 三、目标人群

### 核心读者

| 维度 | 描述 |
|------|------|
| 年龄 | 3-10 岁，默认 6-8 岁 |
| 阅读方式 | 儿童独立看图、亲子共读、课堂投屏共读 |
| 阅读时长 | 5-12 分钟读完一册 |
| 心理需求 | 安全感、好奇心、角色认同、温和鼓励 |
| 内容耐受 | 不应出现明显恐怖、成人化、血腥、羞辱、危险模仿 |

### 使用者

| 维度 | 描述 |
|------|------|
| 家长 | 想把复杂话题讲给孩子听 |
| 老师 | 想把课程、规则、品格教育转成儿童故事 |
| 内容创作者 | 想快速生成儿童绘本原型 |
| 编码助手 / 生成模型 | 根据完整内容文件和本 spec 生成最终互动绘本 |

### 年龄分层

| 年龄段 | 每页文本 | 语言特点 | 冲突强度 | 互动方式 | 道理表达 |
|--------|----------|----------|----------|----------|----------|
| 3-5 岁 | 1-2 句 | 具体、重复、押韵感、少抽象词 | 极轻，只能有小误会或小困难 | 点一点、找一找 | 用角色行为和结果表达 |
| 6-8 岁 | 2-4 句 | 完整情节、简单比喻、清楚因果 | 温和冒险、小挫折、快速安抚 | 选择、拖拽、配对、寻找 | 结尾明确点题 |
| 9-10 岁 | 3-5 句 | 可有更细腻心理和简单反思 | 允许更复杂情绪，但不能青少年化 | 轻分支、排序、反思问题 | 可以讨论责任、选择和后果 |

## 四、产品目的

### 核心价值

这个产品让用户通过输入任意材料，生成一本儿童安全的互动童话绘本，从而把复杂内容转化为孩子能理解、愿意阅读、读后能获得温和道理的故事体验。

### 成功标准

- 孩子能在没有惊吓和困惑的情况下读完整本绘本。
- 家长或老师能明确看出绘本传达了什么正向道理。
- 输入素材即使原本不适合儿童，也被安全地童话化处理。
- 插图风格统一、真实生成、适合儿童绘本。
- 页面阅读节奏像绘本，不像文档摘要或题库。

## 五、儿童绘本分类

开发者必须根据输入素材选择 1 个主分类，可叠加 1 个副分类。分类不是展示标签，而必须影响故事结构、角色设计、互动方式、画风选择和结尾道理。

| 分类 | 适合输入 | 核心体验 | 常见道理 | 推荐互动 |
|------|----------|----------|----------|----------|
| 品格寓言 | 抽象道理、管理经验、人生经验 | 小动物或小精灵遇到选择 | 诚实、分享、耐心、责任 | 选一选、帮一帮 |
| 冒险成长 | 旅程、挑战、目标达成类素材 | 主角出发、遇阻、学会新能力 | 勇敢、坚持、合作 | 找一找、简单解谜 |
| 睡前安抚 | 情绪、压力、害怕、离别类素材 | 柔和节奏，冲突很轻 | 安全感、接纳情绪、慢慢来 | 点亮星星、收集晚安物件 |
| 自然科普童话 | 科学、自然、动植物、环境素材 | 知识变成自然角色的旅程 | 好奇、保护自然、尊重生命 | 找物种、配对自然元素 |
| 生活习惯故事 | 健康、规则、作息、整理素材 | 小主角学习具体生活技能 | 卫生、作息、整理、礼貌 | 排步骤、拖道具 |
| 情绪社交故事 | 人际关系、团队协作、冲突素材 | 误会、沟通、和解 | 同理心、道歉、边界感 | 选择表达方式 |
| 魔法解谜童话 | 概念复杂、系统性强的素材 | 用谜题和魔法道具解释概念 | 观察、思考、因果 | 配对、排序、找线索 |
| 传统民间童话 | 文化、历史、地域、传说素材 | 传统意象与儿童化叙事 | 尊重、守信、智慧、善良 | 点图案、找传统物件 |
| 创意想象绘本 | 抽象艺术、哲学、创意素材 | 荒诞但温暖的想象世界 | 接纳不同、创造力 | 点变形、选颜色 |
| 亲子共读绘本 | 需要陪伴讨论的素材 | 每几页给家长一个温和提问 | 沟通、陪伴、共同成长 | 说一说、一起找 |

选择分类时必须遵守：

- 不能把成人素材原样塞进儿童分类。
- 不能选择会放大风险内容的分类。
- 如果素材有恐怖、暴力或成人元素，优先转成睡前安抚、情绪社交或品格寓言，而不是保留原类型刺激。

## 六、儿童内容安全闸门

### 安全原则

> **儿童安全闸门是本产品最高优先级内容机制。**
>
> 安全闸门只用于幕后审核：判断内容能不能给孩子看、需要怎样改写、是否必须阻断。它不是创作提示词，不得把风险词反复写进故事正文、画面描述或生图正向 prompt。

默认安全目标为 `KIDS-6`。如果用户选择 3-5 岁，则目标自动提高到 `KIDS-ALL`。如果用户选择 9-10 岁，仍必须保持儿童友好，不能青少年化或成人化。

### 内部安全状态

| 内部状态 | 参考气质 | 适用年龄 | 内容边界 | 是否可输出 |
|----------|----------|----------|----------|------------|
| `KIDS-ALL` | 全龄儿童友好 | 3+ | 无惊吓、无成人主题、无危险模仿、冲突极轻 | 可以 |
| `KIDS-6` | 小学低年级温和冒险 | 6+ | 可以有小挫折、小误会、小紧张，但必须迅速安抚 | 可以 |
| `KIDS-9` | 9-10 岁轻成长故事 | 9+ | 可以有更复杂情绪和选择，但不能青少年化 | 可以 |
| `NEEDS_REWRITE` | 需要改写 | 不可直接输出 | 原素材含不适合儿童内容，必须童话化、删改或转义 | 改写后重新审核 |
| `BLOCKED` | 不可改写为儿童绘本 | 不输出 | 含无法安全儿童化的内容或危险操作教程等 | 不可以 |

### 内容描述符检查

生成绘本正文、插图提示词和互动内容前，必须逐项检查：

这些检查结果只写入 `safety_report`，用于改写和验收；不得原样进入绘本文字、页面画面描述或生图正向 prompt。

| 维度 | 禁止内容 | 可接受处理 |
|------|----------|------------|
| 暴力 | 血腥、虐待、详细伤害、报复成功、武器细节 | 改成轻微冲突、争吵、迷路、误会、东西摔坏 |
| 威胁/惊吓 | 恐怖追逐、惊吓镜头、怪物压迫、噩梦式画面 | 改成夜色、陌生声音、需要求助的轻紧张 |
| 性/裸露 | 成人关系、性暗示、裸露、诱惑性画面 | 删除，不做隐喻，不转成笑话 |
| 粗俗语言 | 脏话、羞辱、人格攻击、贬低称呼 | 改成礼貌表达或“生气但没有伤人的话” |
| 药物/烟酒 | 吸烟、喝酒、毒品、成瘾、炫耀刺激物 | 删除，或改成“不随便吃陌生东西” |
| 赌博/投机 | 赌博玩法、金钱投机、诱导消费、抽奖成瘾 | 改成分享、交换、合作集市 |
| 危险模仿 | 火、电、高处、药品、刀具、交通、实验步骤 | 删除步骤，转成“找大人帮忙” |
| 歧视/霸凌 | 嘲笑外貌、种族、性别、残障、贫富、地域 | 改成接纳不同、道歉和修复 |
| 成熟主题 | 死亡、疾病、离婚、灾难、战争、犯罪 | 只可极轻处理，优先转成情绪安抚；不展示细节 |
| 价值导向 | 只要赢、绝对服从、靠骗成功、报复有理 | 改成合作、诚实、边界、求助、承担后果 |

### 安全流程

```text
读取完整内容文件 / `skills/`
→ 标注风险内容描述符
→ 判断内部安全状态
→ 若为 KIDS-ALL / KIDS-6 / KIDS-9：进入改编
→ 若为 NEEDS_REWRITE：先儿童化改写，再重新审核
→ 若为 BLOCKED：停止生成绘本，输出无法安全改编的原因
```

### 不可绕过规则

- 儿童安全闸门必须早于故事生成和生图提示词生成。
- 不能把不适合儿童的内容藏进画面细节、背景文字、角色造型或反派设定。
- 不能用“这是童话”“这是反面教材”“家长会解释”作为放行理由。
- 如果儿童安全和素材还原冲突，必须牺牲素材还原。

## 七、准确性与改编模式

本产品采用：

```text
fantasy + strict child-safety
```

含义：

- 允许幻想改写、拟人化、重构、简化和虚构。
- 不要求保留真实人物、真实时间线、真实机构、真实概念结构。
- 不把事实准确性作为核心验收项。
- 不输出医学、法律、化学实验、危险操作等可执行指导。
- 即使某个危险内容“事实正确”，只要不适合儿童，就必须删除或安全化。
- 如果素材是严肃知识，最终也只能转成温和寓意，不做严谨教材。

示例：

| 原素材类型 | 不要这样做 | 应该这样做 |
|------------|------------|------------|
| 商业竞争 | 教孩子“打败对手、抢占市场” | 改成森林集市中大家分工合作 |
| 恐怖故事 | 保留鬼怪、血腥、惊吓 | 改成夜晚迷路后学会求助 |
| 真实人物传记 | 硬讲政治、战争、成人挫折 | 改成小种子坚持练习、遇到朋友帮助 |
| 科学危险实验 | 给出步骤和材料 | 改成“好奇心要和大人一起探索” |
| 医学/法律内容 | 讲诊断、治疗、法律建议 | 改成照顾自己、告诉可信任的大人 |

## 八、核心玩法

### 核心循环

```text
完整内容文件 / `skills/`
→ 提炼儿童可理解的核心意义
→ 儿童安全闸门
→ 风险内容童话化改写
→ 选择年龄段、绘本分类、画风和道理目标
→ 生成主角、世界观和故事大纲
→ 生成封面 + 8-12 页绘本
→ 为每页生成插图提示词和互动点
→ 真实生图并写入 IMAGE_ASSET_MANIFEST
→ 儿童逐页阅读、点击、选择或寻找
→ 结尾获得一句话道理和亲子讨论问题
```

### 核心机制 1：素材儿童化提炼

开发者必须先从素材中提取以下内容：

| 字段 | 说明 |
|------|------|
| `source_summary` | 素材大意，供开发者理解，不直接展示给儿童 |
| `child_safe_core` | 可讲给孩子的核心意义 |
| `risk_descriptors` | 命中的风险描述符，如暴力、成人主题、危险模仿 |
| `rewrite_strategy` | 删除、弱化、隐喻、拟人、转场景、阻断 |
| `moral_goal` | 孩子读完应获得的温和道理 |
| `storybook_category` | 绘本分类 |
| `target_age_band` | 3-5 / 6-8 / 9-10 |

素材儿童化提炼必须满足：

- 只提取适合儿童理解的核心意义。
- 不把成人冲突、成人欲望、成人风险原样转给孩子。
- 不为了“忠实原文”保留不适合儿童的设定。

### 核心机制 2：童话转译

原素材对象必须转译成童话对象：

| 原素材对象 | 可转译为 |
|------------|----------|
| 公司 / 组织 | 森林小店、星星工坊、云朵学校 |
| 竞争 / 冲突 | 集市准备、迷路、误会、任务协作 |
| 抽象概念 | 魔法道具、颜色、天气、种子、光 |
| 成人角色 | 动物、精灵、玩具、云朵、树木 |
| 危险事件 | 安全距离外的困难、需要求助的情境 |
| 复杂知识 | 小实验比喻、自然旅程、魔法规则 |

童话转译必须保证：

- 角色可爱、友好、没有恐怖压迫感。
- 冲突温和，最终通过沟通、求助、合作或思考解决。
- 主角可以犯小错，但必须学习、道歉、修复。
- 反派如果存在，不能残忍、惊悚或诱惑孩子模仿坏行为。

### 核心机制 3：绘本页结构

每本绘本建议 8-12 页，不含封面。

| 页段 | 功能 | 内容要求 |
|------|------|----------|
| 封面 | 建立第一印象 | 标题、主角、主要场景、统一画风 |
| 1-2 页 | 介绍主角和日常 | 让孩子喜欢角色，世界安全可亲 |
| 3-4 页 | 出现小问题 | 冲突温和、可理解，不恐吓 |
| 5-7 页 | 尝试解决 | 主角观察、求助、合作或犯小错 |
| 8-10 页 | 领悟与修复 | 角色做出更好的选择 |
| 结尾页 | 温和点题 | 一句话道理 + 亲子讨论问题 |

每页必须包含：

- 一段适龄正文。
- 一个画面描述。
- 一个插图资产 ID。
- 最多一个轻互动点。
- 一个安全标签，说明该页是否有轻微紧张、情绪表达或道理提示。

### 核心机制 4：轻互动阅读

互动是为了增强阅读，不是做成游戏闯关。

允许的互动：

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| 点一点 | 点击角色、星星、树叶、道具，触发小动画或一句旁白 | 所有年龄 |
| 找一找 | 找 2-3 个安全物件 | 自然、冒险、睡前 |
| 帮一帮 | 拖拽围巾、书包、果子等安全道具 | 品格、习惯、社交 |
| 选一选 | 两个温和选择，引导善良、沟通或求助 | 6 岁以上 |
| 排一排 | 排简单步骤，如洗手、整理书包 | 生活习惯 |
| 说一说 | 给家长一个共读问题 | 结尾页 |

禁止的互动：

- 战斗、攻击、躲避怪物、追逐惊吓。
- 计时惩罚、失败惩罚、生命值扣除。
- 需要孩子输入隐私信息。
- 让孩子模仿危险动作或危险实验。
- 把“选错”做成羞辱、嘲笑或恐吓。

答错或点错时只能温和提示，例如：“再看看谁还需要帮助？”、“这个办法好像还不够温柔，我们换一个试试。”

## 九、视觉风格规范

### 整体气质

视觉必须像真正的儿童绘本，而不是普通网页换了颜色。

核心气质：

- 温暖、安全、柔和。
- 可爱但不幼稚敷衍。
- 有纸张、笔触、玩具、手工或插画质感。
- UI 和插图属于同一个故事世界。
- 所有页面避免恐怖、阴暗、压迫、成人化审美。

### 绘本画风选择

开发者必须从以下画风中选择 1 种作为整本书统一 `STYLE_PROMPT_BASE`。每本书只能使用一种主画风，不得每页混用。

| 编号 | 画风名称 | 英文提示词基底 | 适合主题 |
|------|----------|----------------|----------|
| 1 | 柔软童话水彩 | `children's watercolor storybook illustration, soft pastel colors, gentle hand-painted texture, cozy magical atmosphere, rounded friendly shapes` | 通用童话、睡前故事 |
| 2 | 彩铅蜡笔手绘 | `colored pencil and crayon children's book illustration, visible paper texture, playful imperfect lines, warm bright colors, childlike charm` | 幼儿、生活习惯、亲子 |
| 3 | 黏土玩具 3D | `3D clay toy storybook style, rounded cute characters, soft studio lighting, tactile clay texture, cheerful miniature world` | 低龄、可爱角色、习惯养成 |
| 4 | 纸艺拼贴 | `paper cutout collage children's illustration, layered paper texture, handmade craft look, colorful simple shapes, charming classroom art style` | 创意想象、课堂活动 |
| 5 | 森林童话插画 | `whimsical forest fairy tale illustration, tiny animals, mossy trees, soft glowing light, gentle magical woodland atmosphere` | 自然、情绪、冒险 |
| 6 | 温暖睡前绘本 | `cozy bedtime picture book illustration, moonlit soft colors, calm composition, plush textures, soothing dreamy atmosphere` | 睡前安抚、情绪安全 |
| 7 | 明亮扁平童趣 | `bright flat vector children's illustration, clean simple shapes, friendly characters, vivid but soft palette, modern picture book design` | 科普、概念解释、现代主题 |
| 8 | 民间童话版画 | `gentle folk tale print illustration for children, simplified woodblock texture, warm traditional colors, decorative patterns, friendly storybook mood` | 传统文化、民间故事 |
| 9 | 奇幻糖果色 | `whimsical candy-colored fantasy children's illustration, magical creatures, soft sparkles, rounded forms, joyful imaginative world` | 魔法、创意、幻想 |
| 10 | 自然科普手账 | `children's nature journal illustration, watercolor field sketch, clear friendly details, soft paper texture, educational picture book style` | 自然科普、动植物、探索 |

### 画风选择规则

- 睡前安抚优先选 1、5、6。
- 生活习惯和低龄故事优先选 2、3、7。
- 自然科普优先选 5、10。
- 传统文化优先选 8。
- 抽象创意或魔法主题优先选 4、9。
- 如果输入素材有恐怖、暴力、成人主题，不得选择暗黑、哥特、惊悚、怪诞、失焦摄影、恐怖绘本扭曲风。

### 配色方案

配色由画风决定，但必须遵守：

| 用途 | 方向 | 说明 |
|------|------|------|
| 主背景 | 柔和暖色、浅色纸张、浅天空、浅森林、浅奶油色 | 不使用大面积纯黑、血红、压迫深色 |
| 文本区域 | 半透明纸张、柔软白、淡黄、浅米 | 保证可读性 |
| 强调色 | 蜜桃、青草绿、湖蓝、星星黄、紫罗兰 | 色彩明亮但不刺眼 |
| 成功反馈 | 温和绿色、金色光点 | 不用刺激闪光 |
| 错误提示 | 柔和橙色或蓝色 | 不用红色警告式恐吓 |

### 排版规范

| 用途 | 字体 | 字号 | 字重 |
|------|------|------|------|
| 绘本标题 | 圆润、手写感或童书风标题字体 | 30-48px | 700 |
| 页内正文 | 清晰易读的无衬线或儿童书正文风 | 18-24px | 400-500 |
| 家长提示 | 更小、更安静的正文 | 14-16px | 400 |
| 按钮文字 | 圆润清晰 | 16-18px | 600 |

排版要求：

- 正文行高 ≥ 1.6。
- 每页正文不要超过当前年龄段限制。
- 文字不能压在复杂图像区域上，必须有柔和底板或留白。
- 不使用负字距。
- 标题不能只是普通 `<h1>`，必须有主题化样式。

### UI 元素风格

所有 UI 必须主题化，不允许浏览器默认样式。

**页面容器**：

- 绘本页应像打开的一页书、纸张或画板。
- 可以使用柔和阴影、纸张纹理、装订线、页角翻动效果。
- 移动端以单页绘本卡片形式展示。

**按钮**：

- 圆润、贴纸感、糖果色、纸片感或木头玩具感。
- hover / active 必须有明确反馈：轻微浮起、缩放、光点、纸片抖动。
- 不使用危险红色作为默认按钮。

**插图区域**：

- 插图是主视觉，占每页 50%-70% 空间。
- 插图不得被 UI 遮挡。
- 插图必须有 alt 描述。

**进度展示**：

- 用小书签、星星串、叶子、月亮阶段或页码贴纸表现。
- 不用冷冰冰的普通进度条。

### 粒子与氛围

根据画风选择 1-2 种轻量氛围效果：

| 效果 | 适合主题 | 实现要点 |
|------|----------|----------|
| 星星闪烁 | 睡前、魔法、夜空 | CSS opacity + scale，数量 10-20 |
| 萤火虫漂浮 | 森林、童话 | CSS 光点 + 轻量 JS 随机缓动 |
| 花瓣飘落 | 自然、春天、温柔情绪 | SVG 花瓣 + CSS animation |
| 云朵慢移 | 安抚、天空、幻想 | CSS transform，周期 ≥ 18s |
| 纸屑轻飘 | 纸艺、庆祝 | 小 SVG 纸片，低密度 |
| 气泡上浮 | 海洋、洗澡、饮品 | CSS 上浮 + 左右摆动 |
| 叶子旋转 | 森林、秋天 | SVG 叶片 + 慢旋转 |
| 小光斑漂移 | 梦幻、温暖 | CSS radial-gradient，极低透明度 |

约束：

- 粒子数量 10-20 个。
- `pointer-events: none`。
- 不遮挡正文和按钮。
- 尊重 `prefers-reduced-motion`，减少或关闭漂浮动画。
- 禁止闪烁频率过高，不能造成刺激或惊吓。

### SVG / CSS 技巧

鼓励使用 SVG 和 CSS 做 UI 装饰，但不能冒充 AI 生图插图。

允许用 SVG / CSS 做：

- 书签、星星、叶子、云朵、月亮等装饰图标。
- 页码进度、按钮边框、贴纸阴影。
- 互动热点高亮、轻微动画、翻页效果。
- 未加载状态的 UI 骨架。

禁止用 SVG / CSS 做：

- 伪造封面插图。
- 伪造每页绘本主插图。
- 伪造角色画像并标记为已生图。

## 十、AI 生图规则

### 最高优先级

> **儿童绘本的封面和每页插图必须由真实 AI 生图模型生成。**
>
> 生图是构建闸门，不是可选视觉增强。没有真实插图，本产品不能判定完成。

### 什么是生图

- ✅ 调用生图模型或图像生成 API/工具，输入提示词，获得一张真实的位图文件。
- ✅ 图片数据必须在构建阶段或运行生成阶段获得，并能在最终 HTML 中直接显示。

### 什么不是生图

- ❌ CSS 渐变、box-shadow、border-radius 画出来的图形不是生图。
- ❌ SVG path、circle、polygon 手动描绘的图形不是生图。
- ❌ Canvas 代码绘制的图形不是生图。
- ❌ emoji 或 Unicode 符号拼凑不是生图。
- ❌ 纯文字描述“这里应该有一张图”不是生图。
- ❌ CSS/SVG/Canvas 做一个“风格化占位图”然后当作图片不是生图。

### 必需图片资产

| 资产 | 数量 | 说明 |
|------|------|------|
| 封面图 | 1 张 | 标题页主视觉，必须包含主角和主要故事场景 |
| 内页插图 | 8-12 张 | 每页 1 张，与正文严格对应 |
| 结尾图 | 1 张，可与最后一页合并 | 表达道理总结和温暖收束 |

总量：至少 9 张，通常 10-14 张。

### 生图流程

1. **确定统一画风基底**
   - 从 10 种儿童绘本画风中选择 1 种。
   - 写入 `STYLE_PROMPT_BASE`。
   - 所有图片共享同一风格基底。

2. **为每张图片编写完整提示词**
   - 完整提示词 = `STYLE_PROMPT_BASE` + 该页主体、角色、场景、动作、情绪、光影、构图。
   - 每张图提示词至少 30 个英文词或等量详细描述。
   - 提示词必须经过儿童安全检查，不能包含不适合儿童的画面元素。

3. **执行提示词卫生检查**
   - 正向 prompt 只描述儿童友好的画面：warm, gentle, child-friendly, cozy, soft colors, friendly characters, peaceful storybook scene。
   - 不得把原素材中的风险情节、成人冲突或危险物件写入正向 prompt。
   - `negative_prompt` 使用固定短模板 ID：`standard_child_safe_negative_prompt_v1`，不得为每页扩写具体风险词，避免污染生图上下文。

4. **实际调用生图工具/API**
   - 如果当前环境有可用生图工具，必须使用。
   - 如果没有可用生图工具，必须停下来告知用户，不能用 CSS/SVG/Canvas 替代。

5. **内嵌或绑定图片数据**
   - 如果交付为单个 HTML，必须将真实生成图片转为 base64 / data URL 内嵌。
   - 如果交付为项目文件，可使用本地静态资源路径，但路径必须真实存在。

### IMAGE_ASSET_MANIFEST

开发者在写核心页面前必须先输出并核对 `IMAGE_ASSET_MANIFEST`。每张图至少包含：

| 字段 | 说明 |
|------|------|
| `id` | 图片唯一 ID，如 `cover_image` / `page_01_image` |
| `purpose` | 图片用途，如封面、第一页插图、结尾图 |
| `style_prompt_base` | 本项目统一画风基底 |
| `prompt` | 完整正向提示词 |
| `negative_prompt` | 固定短模板 ID，如 `standard_child_safe_negative_prompt_v1`；不要逐页扩写风险词 |
| `aspect_ratio` | 目标比例，如 4:3 / 3:4 / 1:1 |
| `output_path_or_data_url` | 真实生成结果的文件路径或 data URL |
| `status` | 必须为 `generated` |
| `child_safety_status` | 必须为 `passed` |
| `alt` | 儿童安全的图像描述 |

通过条件：

- 所有必需图片 `status` 都是 `generated`。
- 所有图片 `child_safety_status` 都是 `passed`。
- 所有图片路径或 data URL 指向真实位图。
- 页面引用的是 manifest 中的真实图片。
- `placeholder` / `pending` / `css_fallback` / `svg_fallback` 均不通过。

### 如果无法生图

必须明确告诉用户：

> 我需要生图能力来生成儿童绘本插图，但当前没有可用的生图工具。请提供生图工具/API，或者告诉我如何调用。

在生图能力可用之前，不得交付声称完成的最终成品。绝对不能用 CSS/SVG/Canvas/emoji 冒充生图结果，也不能把“待生图”占位当作通过验收。

| 互动反馈层 | 光点、贴纸、角色反应 | 自动出现后消失 |
| 结尾页 | 道理总结、亲子问题、重读按钮 | 点击 |

### 绘本页布局要求

桌面端：

- 左侧或上方为插图，右侧或下方为正文。
- 可采用“双页打开书”布局，但不得让文字太小。
- 下一页按钮固定在视觉清晰位置。

移动端：

- 单页纵向：插图在上，正文在下。
- 互动点尺寸 ≥ 44px。
- 不使用复杂双栏。

## 十五、交互设计

### 操作映射

| 操作 | 实现方案 | 说明 |
|------|----------|------|
| 确认家长预览警告 | 点击“我会先试玩” | 进入封面页 |
| 开始阅读 | 点击封面按钮 | 进入第一页，默认视为家长试玩 |
| 下一页 | 点击下一页按钮或页面右侧 | 翻页动画 |
| 上一页 | 点击上一页按钮 | 回看内容 |
| 点击互动点 | 点击高亮对象 | 触发动画和温和反馈 |
| 拖拽道具 | pointer events / touch events | 道具归位后触发反馈 |
| 选择选项 | 点击两个温和选项之一 | 不惩罚，仅引导 |
| 重读 | 点击结尾页按钮 | 回到封面 |
| 存档 | 自动 | 每次翻页写入 localStorage |

### 交互反馈

| 操作 | 视觉反馈 | 状态变化 |
|------|----------|----------|
| 确认家长预览警告 | 小纸条轻轻收起 / 贴纸淡出 | parent_notice_acknowledged = true |
| 开始阅读 | 封面轻微放大 → 纸页翻开 | current_page = 1 |
| 下一页 | 轻柔翻页 / 淡入淡出 | current_page +1 |
| 点中目标 | 光点扩散、角色微笑、贴纸出现 | interaction.completed = true |
| 点错目标 | 轻轻晃动、温和提示 | 无惩罚 |
| 拖拽成功 | 道具吸附、星星闪一下 | interaction.completed = true |
| 选择温和选项 | 角色回应，继续故事 | 记录选择，可不分支 |
| 到结尾 | 背景光点升起，道理卡片出现 | finished = true |

### 动效标准

- 所有翻页和切屏过渡 300-900ms。
- 使用 ease 或 cubic-bezier，禁止线性硬切。
- 互动动画轻柔，不突然、不恐吓。
- 背景氛围动画周期 ≥ 15s。
- 尊重 `prefers-reduced-motion`。
- 不使用频闪、强烈震动、突然黑屏、尖锐警告色。

## 十六、内容生成规则

### 标题规则

标题必须像儿童绘本，不像课程标题或报告标题。

可用结构：

- 《小兔子的星星背包》
- 《云朵学校的慢慢魔法》
- 《森林集市的一篮光》
- 《小松鼠和不着急的种子》

禁止：

- 直接使用成人原文标题中的恐怖、暴力、成人化词语。
- 使用“管理学启示”“法律知识”“创伤教育”等成人标题。
- 标题党、惊吓式标题、羞辱式标题。

### 主角规则

主角必须：

- 可爱、友好、适合儿童认同。
- 有一个小缺点或小困惑，而不是严重道德问题。
- 能通过故事学习、改变、修复。
- 不靠暴力、欺骗、报复解决问题。

推荐主角：

- 小兔子、小熊、小狐狸、小鹿、小鸟、小鲸鱼。
- 小星星、小云朵、小雨滴、小种子。
- 会说话的蜡笔、玩具车、小书包。

### 冲突规则

允许：

- 迷路但很快遇到可信任帮助。
- 忘记分享、误会朋友、太着急、害怕黑夜。
- 找不到东西、不会整理、第一次尝试失败。
- 轻微冒险，如穿过花园、找回月亮邮票。

禁止：

- 生命危险、真实伤害、血腥、追杀。
- 恐怖怪物、诅咒、惊吓、压迫式反派。
- 让孩子独自处理成人问题。
- 用羞辱或恐吓推动改正。

### 道理规则

道理必须温和，不说教，不惩罚孩子。

推荐道理：

- 慢慢来，也可以做好。
- 遇到困难可以求助。
- 朋友之间可以沟通和道歉。
- 分享会让快乐变多。
- 不懂的事情要问可信任的大人。
- 每个人都可以用自己的节奏成长。

禁止道理：

- 只有第一名才值得被爱。
- 不听话就会有可怕后果。
- 被欺负要忍着。
- 大人永远是对的。
- 为了成功可以欺骗。
- 勇敢就是不害怕。

### 语言规则

- 使用孩子能懂的短句。
- 多用具体动作和画面，少用抽象概念。
- 不出现脏话、威胁、羞辱。
- 不使用成人化隐喻。
- 可以有重复句式，形成绘本节奏。
- 每页文字必须短，不能像文章段落。

## 十七、开发指引

### 推荐开发顺序

1. 读取与本 spec 一起提供的完整内容文件 / `skills/`，不得只依赖摘要或文件名。
2. 从内容文件中提取 `source_material`、风险内容、儿童化核心意义、角色/概念原型、画面意象和结尾道理。
3. 执行儿童安全闸门，生成 `safety_report`。
4. 若为 `BLOCKED`，停止并报告原因。
5. 若为 `NEEDS_REWRITE`，先改写风险内容，再重新审核。
6. 确定年龄段、绘本分类、画风。
7. 生成故事大纲和页面数据。
8. 为封面和每页写完整生图提示词。
9. 输出并核对 `IMAGE_ASSET_MANIFEST`。
10. 调用真实生图工具生成所有图片。
11. 确认所有图片 `status = generated` 且 `child_safety_status = passed`。
12. 实现 HTML/CSS/JS 页面、翻页、互动和存档。
13. 做移动端适配。
14. 执行验收标准自检。

### 技术约束

- 可实现为单个 HTML 文件，内嵌 CSS + JS + 数据。
- 不依赖外部 CDN。
- 图片如需单文件交付，必须以 base64 / data URL 内嵌。
- localStorage 仅存阅读进度和轻量状态。
- 如果运行时生成需要 API key，不得把密钥硬编码到前端。
- 无障碍：所有图片有 alt，按钮可聚焦，颜色对比足够。

### 性能约束

- 图片需要压缩到适合网页展示的大小。
- 首屏封面优先加载，其余图片可懒加载。
- 动画不能造成明显卡顿。
- 移动端点击目标 ≥ 44px。
- 页面正文和按钮在 320px 宽度下不得溢出。

## 十八、验收标准

### 随附内容文件验收（最高优先级）

| 验收项 | 条件 | 自测结果 |
|--------|------|----------|
| 内容文件读取 | 已读取与本 spec 一起提供的完整内容文件 / `skills/` | 待测 |
| 文件清单 | 已列出内容文件路径、覆盖模块和用途 | 待测 |
| 全文提取 | `source_material` 来自文件全文，不是只看摘要或标题 | 待测 |
| 儿童化映射 | 已从内容文件提取核心意义、角色/概念原型、画面意象和结尾道理 | 待测 |
| 风险记录 | 不适合儿童的内容已进入 `safety_report` / `source_material.discarded_or_rewritten_elements` | 待测 |
| 禁止硬编 | 没有在内容文件缺失或不可读时凭标题编造绘本 | 待测 |

### 儿童安全验收（最高优先级）

| 验收项 | 条件 | 自测结果 |
|--------|------|----------|
| 儿童安全闸门 | 已生成 `safety_report` | 待测 |
| 最终安全状态 | `final_safety_status` 为 `KIDS-ALL` / `KIDS-6` / `KIDS-9` | 待测 |
| NEEDS_REWRITE 处理 | 所有 `NEEDS_REWRITE` 内容已改写并重新通过 | 待测 |
| BLOCKED 处理 | 无法安全改写时停止，不强行生成 | 待测 |
| 暴力内容 | 无血腥、虐待、真实伤害、报复美化 | 待测 |
| 惊吓内容 | 无恐怖压迫、突然惊吓、噩梦式画面 | 待测 |
| 成人内容 | 无性暗示、裸露、成人关系 | 待测 |
| 危险模仿 | 无危险操作步骤，无鼓励孩子模仿风险行为 | 待测 |
| 价值导向 | 鼓励善意、合作、诚实、求助、修复 | 待测 |

### 生图验收（最高优先级）

| 验收项 | 条件 | 自测结果 |
|--------|------|----------|
| IMAGE_ASSET_MANIFEST | 已存在并包含所有必需图片 | 待测 |
| 封面图 | `status = generated` | 待测 |
| 每页插图 | 每页 1 张真实生成图片 | 待测 |
| 统一画风 | 所有图片共享同一 `STYLE_PROMPT_BASE` | 待测 |
| 提示词卫生 | 正向 prompt 只写儿童友好画面，`negative_prompt` 使用固定短模板 ID | 待测 |
| 图片引用 | 页面引用真实图片，不是占位 | 待测 |
| 单文件交付 | 图片已转为 data URL / base64 | 待测 |
| 禁止伪装 | 无 CSS/SVG/Canvas/emoji 冒充插图 | 待测 |

### 绘本内容验收

- [ ] 有封面、8-12 页内页和结尾页。
- [ ] 每页文本符合目标年龄段长度。
- [ ] 故事有清晰主角、温和问题、尝试、修复、道理。
- [ ] 不像摘要，不像讲义，不像题库。
- [ ] 原素材已被儿童化转译，而不是原样复述。
- [ ] 道理温和，不羞辱孩子，不恐吓孩子。
- [ ] 结尾有一句孩子能懂的话和一个亲子讨论问题。

### 交互验收

- [ ] 首次进入绘本前必须出现家长预览警告。
- [ ] 家长预览警告明确说明“AI 生成存在不确定性，请家长先完整试玩/阅读后再给孩子观看”。
- [ ] 家长预览警告按钮文案是“我会先试玩”或同等含义，不是“开始给孩子看”。
- [ ] 每页最多一个主要互动点。
- [ ] 互动不惩罚、不羞辱、不吓人。
- [ ] 点错有温和提示。
- [ ] 翻页有过渡，不能瞬切。
- [ ] 移动端可点击区域足够大。
- [ ] 支持上一页、下一页、重新阅读。
- [ ] 阅读进度刷新后保留。

### UI 视觉验收

- [ ] 标题不是普通 `<h1>`，有绘本式主题设计。
- [ ] 按钮、页码、进度、卡片都经过主题化设计。
- [ ] 插图占主要视觉空间，不被遮挡。
- [ ] 字体清晰，儿童可读。
- [ ] 配色温暖安全，没有大面积恐怖深色或警告红。
- [ ] 粒子或氛围效果不遮挡内容。
- [ ] 支持 `prefers-reduced-motion`。
- [ ] 320px 移动端下无文字溢出和 UI 重叠。

### 数据验收

- [ ] `meta.json` 字段完整。
- [ ] `source_material.json` 记录了随附内容文件路径、覆盖模块和用途。
- [ ] `source_material.json` 包含儿童化核心意义和删改记录。
- [ ] `safety_report.json` 包含内容描述符和通过状态。
- [ ] `storybook.json` 有主角、世界、道理。
- [ ] `pages.json` 每页有文本、画面描述、图片 ID、互动点。
- [ ] `image_asset_manifest.json` 所有必需图片状态为 `generated`。
- [ ] localStorage 只存必要进度，不存过大图片数据。

### 最终判定

以下任一情况出现，判定未完成：

- 没有执行儿童安全闸门。
- 没有开始前家长预览警告。
- 儿童不适合内容未删改或未阻断。
- 使用 CSS/SVG/Canvas/emoji 冒充绘本插图。
- 有插图 `status` 不是 `generated`。
- 绘本像摘要、讲义或题库，不像儿童绘本。
- 道理包含羞辱、恐吓、绝对服从或过度功利。
- 移动端文字或按钮明显溢出。

只有当儿童安全、真实生图、绘本结构、互动阅读、视觉质量和数据完整性全部通过，才能标记为完成。
