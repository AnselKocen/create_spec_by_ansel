|------|--------|------|

| 强调色（玩家/关键元素） | {{COLOR_ACCENT}} | 例：#e94560 荧光红 |

> 配色原则：
> - **配色必须根据游戏主题设计**——开发者应先分析游戏的世界观和氛围，再决定色彩方向，不可套用默认色板
> - 主色奠定氛围，强调色突出玩家和核心交互，辅助色区分不同游戏对象
> - 文本与背景对比必须要高

### 必须实现的视觉效果

| 游戏事件 | 视觉效果要求 |
|---------|------------|
| 玩家受伤/死亡 | 画面短暂红闪 + 角色抖动或爆炸粒子 |
| 通关/胜利 | 全屏庆祝动画（粒子爆炸/文字放大/颜色切换） |
| 失败/游戏结束 | 画面变灰或慢动作效果 + 明确的 Game Over UI |
| 拾取道具/得分 | 飘字（+分数）或高亮闪烁 |
| 关键操作（攻击/跳跃等） | 残影、拖尾或短暂特效 |
| 场景/关卡切换 | 淡入淡出或滑动过渡，不能瞬切 |

### 动效标准

- 所有角色移动需有缓动（easing），不能匀速线性
- 按钮/可交互元素必须有 hover/active 状态变化
- 游戏整体帧率目标 ≥ 30fps，不能出现明显卡顿
- 尊重 `prefers-reduced-motion` 系统偏好

## 七、UI 设计与氛围

> 本节提供 UI 设计的技巧和要求。游戏的 UI 不是套默认样式，而是主题的一部分。

### 整体 UI 设计要求

> **游戏的所有界面都是风格的一部分，不只是首页。** 从首页到游戏内到结算页，UI 的质感必须统一，必须与选定的艺术风格一致。不能首页很精致，进了游戏就变成默认样式。

- 所有 UI 元素（按钮、面板、弹窗、提示文字等）必须经过主题化设计，禁止使用浏览器默认样式
- UI 的形状、颜色、材质感应与艺术风格呼应（如像素风格用方块按钮、玻璃拟态用模糊半透明面板、手绘风用不规则边缘等）
- 游戏内界面和首页应有视觉连续性——玩家不会觉得"进了另一个网页"
- 信息展示要有设计感，不是把数据堆在屏幕角落

### 首页标题设计

> **游戏标题不是普通的 `<h1>` 文字。** 标题是玩家看到的第一个元素，必须有主题化的设计。

- 字体效果必须与游戏主题匹配（用 CSS 实现：`text-shadow` 发光、`background-clip: text` 渐变、`-webkit-text-stroke` 描边、`letter-spacing` 调节等）
- 可加入 SVG 装饰元素围绕标题（如藤蔓、星星、齿轮、像素边框等，取决于主题）
- 标题可有入场动画（渐显、缩放、逐字出现等）
- 副标题/tagline 用更小更轻的字体，与主标题形成层次

### 首页背景图生成（必须实现）

> **首页必须有一张与游戏主题匹配的背景图。** 由 AI 自行编写提示词并调用生图模型生成。

**什么是"生图"：**
- ✅ 调用生图模型（如图像生成 API / 工具），输入提示词，获得一张真实的图像文件
- ❌ 用 CSS 渐变、box-shadow 画出来的图形 — **这不是生图**
- ❌ 用 SVG path 手动描绘 — **这不是生图**
- ❌ 用 Canvas 代码绘制 — **这不是生图**

**要求**：
- 根据游戏主题编写生图提示词，生成一张氛围背景图
- 背景图仅用于首页/标题页，游戏内界面不使用此背景图
- 背景图作为最底层图层（z-index 最低），UI 元素浮在其上
- 背景图整体偏暗或加半透明遮罩，确保前景文字和按钮清晰可读

**游戏内图片生成（可选但鼓励）**：
- 在 AI 认为有必要时（如关卡过渡画面、胜利/失败场景、关键剧情节点），可以生成额外图片
- 所有生成的图片必须风格统一

### 首页粒子/氛围效果（必须实现）

> **首页不能是"一个标题 + 一个按钮"的白板。** 必须有与主题匹配的氛围粒子或装饰效果。

根据游戏主题选择 1-2 种：

| 效果 | 适用主题 | 实现要点 |
|------|---------|---------|
| 花瓣飘落 | 自然/春天/日系/田园 | SVG 花瓣 + CSS animation，随机起点/速度/旋转 |
| 星星闪烁 | 太空/夜晚/魔法/梦幻 | CSS opacity + scale 脉动，不同周期错开 |
| 雪花飘落 | 冬季/冰雪/北欧 | CSS animation 竖向+微弱横向漂移，大小不一 |
| 气泡上浮 | 水下/海洋/饮品 | CSS animation 上浮 + 轻微左右摆动 |
| 萤火虫漂浮 | 森林/夏夜/童话 | CSS 光点 + JS 随机缓动路径 |
| 灰烬/火星 | 火焰/战争/废墟/锻造 | CSS animation 上浮 + 随机摆动 + opacity 渐隐 |
| 落叶旋转 | 秋天/自然/怀旧 | SVG 叶子 + CSS transform rotate |

| 几何碎片 | 抽象/极简/现代 | CSS 三角形/圆形，缓慢旋转漂浮 |

- 数量 10-30 个，不影响性能
- CSS animation 为主，必要时辅以轻量 JS
- 不应干扰游戏操作
- 尊重 `prefers-reduced-motion`

### SVG / CSS 设计技巧

> 鼓励使用 SVG 和 CSS 绘制游戏元素和 UI，不要只用 Canvas 画一切。

**SVG 适合做的事**：

| 用途 | 技巧 |
|------|------|
| 游戏角色/对象 | `<path>` + `<circle>` + `<polygon>` 组合，CSS transform 做动画 |
| UI 图标 | 内联 SVG，`currentColor` 继承主题色 |
| 装饰边框 | SVG `<path>` 画不规则边框（锯齿、波浪、藤蔓等） |
| 进度条/血条 | SVG `<rect>` + `stroke-dasharray` 动画 |
| 关卡地图 | SVG 路径 + 节点，CSS 控制状态 |
| 背景纹理 | SVG pattern 重复图案（网格、波纹、六边形等） |
| 粒子形状 | SVG 做主题化粒子模板（星星、花瓣等） |

**CSS UI 设计技巧**：

| 技巧 | 做法 | 避免 |
|------|------|------|
| 主题化按钮 | `clip-path` 裁剪非矩形（六边形、菱形等） | 全部圆角矩形 |
| 发光效果 | `box-shadow` 多层叠加 + 同色不同透明度 | 纯色实心阴影 |
| 渐变质感 | `linear-gradient` / `radial-gradient` 模拟材质 | 纯色平涂 |
| 文字装饰 | `text-shadow` 多层 + `background-clip: text` 渐变 | 普通颜色文字 |
| 状态反馈 | `transition` + `transform: scale()` + 颜色变化组合 | 只变颜色 |
| 毛玻璃面板 | `backdrop-filter: blur()` + 半透明背景 | 纯色不透明面板 |
| 柔和分隔 | `box-shadow: inset` + 微渐变代替实线边框 | 1px solid 分隔一切 |
| 动态背景 | `background-position` 动画做缓慢移动纹理 | 纯色静态背景 |

| 维度 | 选择 |
|------|------|

### 需要持久化的数据
