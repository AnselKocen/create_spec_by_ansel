# 儿童童话绘本生成器玩法方案

## 1. 本次产品方向

产品暂定名：`童话小书匠`

产品类型：`storybook-adapter`，即“任意输入内容转儿童童话绘本”的生成型互动产品。

核心目标：无论用户输入的是书籍、知识材料、故事梗概、人物传记、课程内容、产品说明还是抽象主题，系统都先提炼其中适合儿童理解的核心主题，再改写成一册适合孩子阅读的童话绘本。它不是照搬原文，也不是摘要工具，而是把输入内容重构成“可爱角色 + 清晰冲突 + 温和冒险 + 正向道理 + 绘本插图”的儿童阅读体验。

## 2. 核心设计理念

这是一个“儿童绘本改编器”，不是普通文本改写器。

输入材料可以很复杂、成人化、恐怖、商业、学术或抽象，但输出必须始终变成适合儿童的童话绘本。转换重点不是保留原文风格，而是保留可被儿童理解的核心意义，并将其改写为安全、温柔、有想象力、有教育意义的故事。

儿童安全优先级高于素材还原度，也高于事实准确性。这个产品不追求把原材料讲得“对不对、全不全、严不严谨”，而是追求输出一定适合孩子看、不会吓到孩子、不会教坏孩子。只要能保留一个温和的核心意义，素材可以被大幅童话化、隐喻化、拟人化或重构。

## 3. 目标读者与使用者

主要读者：3-10 岁儿童。

主要使用者：家长、老师、儿童内容创作者，或想把复杂内容变成儿童故事的人。

年龄分层建议：

| 年龄段 | 文本长度 | 语言特点 | 互动深度 | 道理表达 |
|---|---:|---|---|---|
| 3-5 岁 | 每页 1-2 句 | 具体、重复、押韵感 | 点一点、找一找 | 用行为结果表达，不讲大道理 |
| 6-8 岁 | 每页 2-4 句 | 完整情节、简单比喻 | 选择、拖拽、配对 | 结尾明确点题 |
| 9-10 岁 | 每页 3-5 句 | 更细腻的心理和因果 | 轻分支、反思问题 | 允许更复杂的价值讨论 |

## 4. 儿童绘本分类

最终 spec 应要求开发者根据输入材料自动选择 1 个主分类，也允许叠加 1 个副分类。分类不能只是标签，而要影响故事结构、角色设计、互动方式和结尾道理。

| 分类 | 适合输入 | 核心体验 | 常见道理 |
|---|---|---|---|
| 品格寓言 | 抽象道理、管理/商业/人生经验 | 小动物或小精灵遇到选择 | 诚实、分享、耐心、责任 |
| 冒险成长 | 旅程、挑战、目标达成类素材 | 主角出发、遇阻、学会新能力 | 勇敢、坚持、合作 |
| 睡前安抚 | 情绪、压力、离别、害怕类素材 | 柔和节奏，冲突很轻 | 安全感、接纳情绪、慢慢来 |
| 自然科普童话 | 科学、自然、环境、动植物素材 | 知识变成自然角色的旅程 | 好奇、保护自然、尊重生命 |
| 生活习惯故事 | 健康、规则、日常习惯素材 | 小主角学习具体生活技能 | 卫生、作息、整理、礼貌 |
| 情绪社交故事 | 人际关系、冲突、团队协作素材 | 角色误会、沟通、和解 | 同理心、道歉、边界感 |
| 魔法解谜童话 | 概念复杂、系统性强的素材 | 用谜题和魔法道具解释概念 | 思考、观察、因果 |
| 传统民间童话 | 文化、历史、传说、地域素材 | 传统意象与儿童化叙事 | 尊老、守信、智慧、善良 |
| 创意想象绘本 | 抽象艺术、哲学、创意素材 | 荒诞但温暖的想象世界 | 接纳不同、创造力 |
| 亲子共读绘本 | 需要陪伴讨论的素材 | 每几页给家长一个温和提问 | 沟通、陪伴、共同成长 |

## 5. 核心循环

```text
输入素材
→ 提炼儿童可理解的核心主题
→ 过滤或改写不适合儿童的内容
→ 选择绘本分类、年龄段、画风和道理目标
→ 生成主角、世界观和故事大纲
→ 生成 8-12 页绘本页面
→ 为每页生成插图提示词和互动点
→ 儿童逐页阅读、点击、选择或寻找
→ 结尾获得温和道理总结和亲子讨论问题
```

## 6. 转换机制

### 机制 1：素材儿童化提炼

系统必须先读取输入材料，提炼：

- 核心主题：这份材料真正想表达什么。
- 儿童可理解的概念：哪些能直接讲，哪些需要比喻。
- 风险内容：哪些不适合儿童原样出现。
- 可童话化对象：人物、机构、观点、事件可转化为什么角色或场景。
- 道理目标：孩子读完应学到什么温和、正向的东西。

示例原则：商业竞争可以转成“森林摊位一起准备集市”；恐怖素材可以转成“夜晚迷路后学会求助和辨认安全”；复杂知识可以转成“魔法学校里的小实验”。

### 机制 2：儿童安全改写

任何输入内容都必须通过儿童安全层。安全层不是简单删除敏感词，而是改写价值导向。

必须保留：

- 善意、合作、诚实、同理心、责任感。
- 角色做错事后可以学习、道歉、修复。
- 冲突通过沟通、求助、合作、思考解决。
- 孩子能理解的因果关系。

必须避免：

- 暴力、恐吓、血腥、虐待、惊吓式表达。
- 欺骗、霸凌、报复、偷窃、作弊被奖励。
- 危险行为被美化，如独自去危险地点、乱吃东西、模仿危险实验。
- 歧视、羞辱、刻板印象、外貌嘲笑。
- 成人化恋爱、性暗示、成瘾、违法行为。
- 用“吓孩子”“羞辱孩子”“绝对服从”作为教育手段。
- 过度功利化的成功观，如只要赢、只要第一、只要变有钱。

### 机制 3：儿童安全闸门

最终 spec 应建立一个独立的儿童安全闸门。它只用于幕后审核：判断内容能不能给孩子看、需要怎样改写、是否必须阻断。它不是创作提示词，不得把风险词反复写进故事正文、画面描述或生图正向 prompt。

建议内部安全状态：

| 内部状态 | 参考气质 | 适用年龄 | 内容边界 |
|---|---|---:|---|
| `KIDS-ALL` | 全龄儿童友好 | 3+ | 无惊吓、无成人主题、无危险模仿、冲突极轻 |
| `KIDS-6` | 适合小学低年级的温和冒险 | 6+ | 可以有小挫折、小误会、小紧张，但必须迅速安抚 |
| `KIDS-9` | 适合 9-10 岁的轻成长故事 | 9+ | 可以有更复杂情绪和选择，但不能青少年化 |
| `NEEDS_REWRITE` | 需要改写 | 不可直接输出 | 原素材含不适合儿童内容，必须童话化、删改或转义 |
| `BLOCKED` | 不可改写为儿童绘本 | 不输出 | 含无法安全儿童化的内容或危险操作教程等 |

默认安全目标应为 `KIDS-6`，但输出内容必须能向下兼容：即使 6-8 岁孩子独立阅读，也不应看到让家长明显担心的内容。

幕后安全判断维度必须包含：

这些判断结果只写入安全报告，用于改写和验收；不得原样进入绘本文字、页面画面描述或生图正向 prompt。

- 暴力/威胁/惊吓强度。
- 性/裸露/成人关系。
- 粗俗语言和侮辱性表达。
- 药物、酒精、烟草、成瘾行为。
- 赌博、金钱投机、过度消费诱导。
- 危险行为和可模仿风险。
- 歧视、霸凌、羞辱、仇恨。
- 悲伤、死亡、离别、疾病等成熟主题的表达强度。
- 价值导向是否鼓励善意、合作、诚实、求助和修复。

安全闸门通过后才能生成绘本正文和插图提示词。未通过时必须先改写，不能把风险内容交给画面或隐晦文本继续呈现。

### 机制 4：开始前家长预览警告

最终产品必须在进入封面或第一页之前显示一个可爱但明确的家长预览警告。这是固定 P0 模块，不是可选说明。

固定含义：

> 给大人的小提示：本绘本由 AI 生成，故事、画面和互动可能存在一定不确定性。请家长先完整试玩/阅读，确认适合孩子后，再给孩子观看。

要求：

- 视觉可以像贴在绘本前的小纸条、书签或贴纸。
- 主按钮建议为“我会先试玩”，不要写成“开始给孩子看”。
- 家长确认后进入封面页。
- 它不能替代儿童安全闸门，只是额外提醒 AI 生成内容存在不确定性。

### 机制 5：绘本页结构

每本绘本建议 8-12 页，不做过长故事。

| 页段 | 功能 | 内容要求 |
|---|---|---|
| 封面 | 建立第一印象 | 标题、主角、主要场景 |
| 1-2 页 | 介绍主角和日常 | 让孩子喜欢角色 |
| 3-4 页 | 出现小问题 | 冲突温和、可理解 |
| 5-7 页 | 尝试解决 | 主角观察、求助、合作或犯小错 |
| 8-10 页 | 领悟与修复 | 角色做出更好的选择 |
| 结尾页 | 温和点题 | 一句话道理 + 亲子讨论问题 |

### 机制 6：轻互动阅读

互动不能抢走绘本阅读本身。每页最多 1 个主要互动点。

可用互动：

- 点一点：点击角色、星星、树叶、道具，触发小动画或一句旁白。
- 找一找：在画面里找 2-3 个安全物件。
- 帮一帮：拖拽物品给角色，如把围巾给冷的小兔子。
- 选一选：两个温和选择，鼓励孩子选择善良、沟通或求助。
- 排一排：把简单步骤排成正确顺序，如洗手、整理书包。
- 说一说：结尾给家长一个开放提问，不要求孩子输入长文本。

不使用失败惩罚。答错或点错时只给温和提示，例如“再看看谁还需要帮助？”。

用户要求至少 8 种适合儿童绘本的画风。最终 spec 建议内置 10 种，开发者根据输入主题选择 1 种作为整本书统一 `STYLE_PROMPT_BASE`，不能每页乱换风格。

| 编号 | 画风名称 | 英文提示词基底 | 适合主题 |
|---:|---|---|---|
| 1 | 柔软童话水彩 | `children's watercolor storybook illustration, soft pastel colors, gentle hand-painted texture, cozy magical atmosphere, rounded friendly shapes` | 通用童话、睡前故事 |

| 6 | 温暖睡前绘本 | `cozy bedtime picture book illustration, moonlit soft colors, calm composition, plush textures, soothing dreamy atmosphere` | 睡前安抚、情绪安全 |

## 8. 生图模式建议

本产品强烈建议选择“需要真实 AI 生图”。

原因：儿童绘本的核心奖励就是每一页插图。如果没有真实插图，只靠 CSS/SVG/文字无法成立为绘本。

最终 spec 中应写入强约束：

- 生图是构建闸门，不是视觉增强。
- 每页插图和封面图都必须真实调用生图模型生成。
- CSS 渐变、SVG、Canvas、emoji、文字占位都不能冒充插图。
- 每本书必须有统一 `STYLE_PROMPT_BASE`。
- 每张图必须进入 `IMAGE_ASSET_MANIFEST`。
- 如果无法生图，开发者必须停下报告，不能交付假绘本。

待确认：是否正式采用“每页都必须生图”的模式。我的建议是采用。

## 9. 准确性与儿童安全模式建议

默认建议：`fantasy + strict child-safety`，即“幻想改写 + 严格儿童安全”。

原因：这个产品的第一目标不是知识准确，而是把任何输入材料变成适合孩子阅读的童话绘本。内容对不对、是否完整还原、是否符合原书逻辑，都不是最高优先级。最高优先级是：绝对不能出现孩子不应该看的东西，绝对不能输出会教坏孩子、吓到孩子或诱导孩子模仿危险行为的内容。

最终 spec 应采用以下规则：

- 整体叙事：允许大幅幻想化、拟人化、改编、简化，不要求事实严格。
- 输入素材：只提取“适合儿童理解的核心意义”，不强制保留事实细节。
- 科学、医学、法律、危险操作等内容：不做严谨教学，不输出可执行指导；如果涉及风险，转成“安全习惯 / 求助成人 / 不随便模仿”的童话寓意。
- 真实人物、历史事件：可以改成虚构角色和童话王国，不要求还原真实时间线。
- 任何可能引导危险模仿的内容，即使事实正确，也必须删除或安全化。
- 内容必须通过内部儿童安全闸门，目标为 `KIDS-6` 或更安全。

待确认：是否采用这个“准确性不重要，但儿童安全闸门严格”的模式。

## 10. 页面与功能结构

建议核心路径：

```text
素材输入页
→ 生成设置页（年龄段 / 绘本分类 / 画风 / 道理目标 / 生图确认）
→ 安全改写预览页（展示核心主题、风险处理、故事大纲）
→ 家长预览警告页（AI 生成不确定性提示）
→ 绘本阅读页（封面 + 8-12 页）
→ 道理总结页（孩子版一句话 + 家长共读问题）
→ 重新生成 / 换画风 / 调整年龄段
```

关键功能：

| 优先级 | 功能 | 说明 |
|---|---|---|
| P0 | 素材输入 | 支持粘贴文本或由实现者读取指定素材 |
| P0 | 儿童安全改写 | 输出前必须先过滤和重构风险内容 |
| P0 | 家长预览警告 | 提醒 AI 生成内容有不确定性，家长需先完整试玩/阅读 |
| P0 | 绘本分类选择 | 自动推荐，也允许用户手动选择 |
| P0 | 年龄段适配 | 控制语言长度、冲突强度和互动难度 |
| P0 | 8-12 页绘本生成 | 每页包含正文、画面描述、互动点、道理映射 |
| P0 | 真实插图生成 | 封面 + 每页插图必须真实生图 |
| P0 | 亲子共读结尾 | 输出温和道理和讨论问题 |
| P1 | 换画风重生成 | 保持故事结构，仅重生成视觉提示词和图片 |
| P1 | 家长安全报告 | 简短展示删改了哪些不适合儿童的内容类型 |

## 11. 数据对象草案

最终 spec 可围绕这些数据对象展开：

- `source_material`：输入材料、来源名称、核心主题。
- `adaptation_profile`：年龄段、分类、画风、道理目标、准确性模式、目标安全状态。
- `safety_report`：风险内容类型、内部安全状态、处理方式、是否通过儿童安全检查。
- `storybook`：标题、副标题、主角、世界观、页数、结尾道理。
- `page`：页码、正文、画面描述、互动点、插图 prompt、图片资产 ID。
- `interaction`：互动类型、目标物、成功反馈、温和提示。
- `image_asset`：资产 ID、用途、风格基底、完整 prompt、negative prompt、比例、生成状态。
