### ⚠️ 核心设计理念

> **这是一个用第一人称叙事制造恐惧的游戏，不是配了恐怖背景音的电子书。**
>
> 恐怖感来自三个层面的配合：
> 1. **文字**——故事本身，第一人称的沉浸感，让玩家觉得"这事发生在我身上"
> 2. **视觉**——UI 的阴森质感 + 关键场景的 AI 生成 CG 插图，让玩家"看见"恐怖
> 3. **机制**——选择暗中影响命运，玩家在不知情中走向不同结局，事后回想才觉得后怕
>
> 去掉任何一层，恐怖感就打折。特别是视觉——**没有 CG 插图的恐怖游戏不完整**。

### 恐怖子类型选择

> **开发者必须根据素材内容，从以下子类型中选择 1-2 个作为本游戏的恐怖基调。** 不可笼统地写"恐怖"，必须明确是哪种恐怖。

| 子类型 | 核心恐惧 | 特征与常见手法 |
|--------|---------|--------------|
| 心理恐怖 | 怀疑自己的认知/记忆/理智 | 叙述者不可靠，现实在动摇；记忆与证据矛盾；同一场景每次重访细节不同；角色指出"你"没做过的事但有证据；日记/录音与记忆冲突；分不清幻觉还是现实；结局揭示主角一直在被自己欺骗 |
| 宇宙恐怖 | 人类的渺小和无意义 | 无法理解的存在，知道越多越恐惧；研究越深精神越崩溃；看到不该看的东西后世界观碎裂；古老文献中的知识让人发疯；试图理解的人全部消失或变了；真相不是"有怪物"而是"人类根本不重要"；恐惧来自认知而非威胁 |
| 民俗恐怖 | 古老规矩/仪式/禁忌 | 不能打破的规则，打破了就回不去；村庄/社区有外人不知道的"规矩"；规矩看似荒谬但所有人严格遵守；违反规矩的人以诡异方式消失；仪式细节背后藏着真正恐怖的原因；"我们一直这样做"是最可怕的解释；主角发现自己也是仪式的一部分 |
| 身体恐怖 | 自己的身体在变化/不受控制 | 变异、寄生、不可逆的改变；身上出现不该有的东西（多余的痕迹/组织/感知）；照镜子发现细微但确定的变化；身体做了"你"没有下达指令的动作；吃东西的口味/需求开始异常；变化过程缓慢到你无法确定起点；到接受的那一刻才是最恐怖的 |
| 日常侵蚀 | 熟悉的事物逐渐变得不对 | 从正常开始，一点点偏移，到回过神来已经太晚；家具位置微妙变化但没人动过；日常对话中出现不该知道的信息；时间出现无法解释的空白；每天通勤路上某个细节在变化但说不清是什么；周围的人对异常毫无反应；回头看第一章才发现当时已经不对了 |
| 社会恐怖 | 集体中隐藏的恶意/荒谬规则 | 所有人都觉得正常，只有你觉得不对；集体笑着做可怕的事；质疑规则的人被温和地"纠正"；表面友善实则不可退出的社群；大家用日常语气讨论荒谬的事；你开始怀疑是不是自己才是错的；最终发现你也在不知不觉中参与了 |
| 谋杀悬疑 | 凶手就在身边/真相比凶案更可怕 | 一桩命案拉开序幕，线索指向身边的人；每个人都有不在场证明但都有破绽；调查越深发现死者并非无辜；凶手的动机让人脊背发凉；"你"可能不只是调查者而是参与者；真相揭开后才发现还有更深的一层；最恐怖的不是谁杀了人，而是为什么 |
| 困境恐怖 | 逃不出去/封闭空间中的绝望 | 被困在某个地方（建筑/小镇/时间循环）无法离开；出口存在但总是到不了；空间本身在变化或缩小；被困的不只是"你"一个人，同伴逐渐失踪或变化；尝试逃离反而更深陷其中；最终发现被困的原因与自身有关；比起外部威胁，无法逃离本身就是恐怖 |
| 科技恐怖 | 技术失控/数字世界的入侵 | 手机里出现没拍过的照片；AI 开始说不该知道的事；监控录像里有你不记得做过的行为；算法推送越来越精准地戳中恐惧；程序/系统按照某种"意志"在运行；线上与线下的边界在消融；科技的便利背后藏着某种不可逆的交换 |
| 孩童恐怖 | 童年记忆中的阴影/童真的扭曲 | 孩子说出不该知道的事；童谣/儿歌暗藏恐怖含义；玩伴不是真实存在的；童年记忆被发现与事实严重不符；天真的行为背后是可怕的模仿；游戏规则越来越像仪式；"你"小时候做过一件被遗忘但极其可怕的事 |
| 替身/分身恐怖 | 另一个"你"的存在 | 有人看到"你"出现在你没去过的地方；镜中的自己动作慢了半拍；照片里的你表情不对；身边的人说"你昨天不是说过了吗"但你没有；日记里出现不是你写的字迹；另一个"你"在逐渐取代你的生活；分不清哪个才是真的自己 |
| 诅咒/怨念恐怖 | 不可解除的因果报应 | 某个行为触发了不可逆的后果；警告来得太晚，诅咒已经生效；受害者的怨念以某种形式延续；每个试图帮忙的人都被波及；诅咒有规律但无法打破；追溯源头发现起因是一件看似微不足道的事；最恐怖的是"你"可能就是起因 |
| 寄生/操控恐怖 | 意志被夺取/思想不是自己的 | 脑中出现不属于自己的想法；越来越想做一件不合理的事但停不下来；行为模式在向某个人靠拢；"你"开始说出不像自己会说的话；身边的人先后出现同样的变化；抵抗的意志在减弱而且你对此感到安心；寄生者不是要杀你，是要成为你 |

### 反套路约束（强制）
**禁止的元素**：
- ❌ "原来是一场梦"结局

**强制的叙事要求**：
- ✅ 恐怖感必须来自**理解**而非**惊吓**——玩家越想越怕，不是被吓一跳
- ✅ 故事开头必须看起来正常，恐怖是一点点渗透进来的
- ✅ 恐怖源必须与素材内容有内在联系，不是给知识套一个恐怖壳
- ✅ 至少有一个"回头看才发现不对"的设计——前面看似正常的细节，后来才意识到恐怖
- ✅ 第一人称视角，玩家 = 主角，全程用"你"
- ✅ 子类型的特征与手法是**参考方向**，必须结合传入的素材内容进行剧情设计，不可直接套用特征列表中的例子作为故事主线

## 四、视觉风格规范

> 恐怖游戏的视觉是恐怖感的放大器。UI 本身就应该让人感到不安。

### 整体气质

**阴森诡异**：打开页面就应该感到压抑和不安。色调阴暗、文字像是在发抖、界面有衰败和侵蚀的质感。不是"暗黑酷炫"，是"这个页面本身就让人不舒服"。

### 背景系统

> 背景必须由开发者在构建时根据故事氛围，自行编写提示词并调用生图模型生成。

**背景生成要求**：
- 根据故事的核心场景，编写背景图的生图提示词
- 调用生图模型生成一张阴暗压抑的氛围背景图
- 背景图仅用于标题页（首页），其他页面不使用
- 背景图作为最底层图层（z-index 最低），整体偏暗，所有 UI 元素（标题、按钮等）浮在其上方，确保不被遮挡

**背景实现要求**：
- `background-image` 固定铺满视口，`background-size: cover; background-position: center; background-attachment: fixed`
- 背景图上方叠加一层半透明暗色遮罩（偏黑红或偏黑绿，取决于恐怖子类型）
- **可选增强**：遮罩颜色可随章节变化——前期偏灰，后期偏红/偏暗

### UI 元素风格

> UI 是恐怖氛围的一部分，不是中性容器。

**文字区域**：
- 暗色底 + 半透明，边缘有不规则的"侵蚀"感（可用 CSS clip-path 或不规则 border）
- 文字颜色偏灰白，不要纯白——纯白太干净，不够阴森
- 关键恐怖文字可用特殊样式：颜色渐变为暗红、轻微抖动动画、或者字间距异常

**选择按钮**：
- 暗色底 + 极细边框，hover 时边框变红/变亮 + 轻微抖动
- 选择后有短暂的"墨水扩散"或"裂痕蔓延"效果
- 按钮文字可以在 hover 时轻微变形（letter-spacing 微调）

**章节过渡**：
- 画面渐暗 → 黑屏停顿 1-2s → 新章标题缓慢显现（带轻微抖动或故障效果）
- 可加入"重影"效果：标题文字有一个偏移的半透明副本
- 不要华丽，要压抑

**恐怖 UI 特效（CSS 实现）**：
- **重影/故障效果**：文字或图像的 RGB 通道偏移（用多层 text-shadow 或 CSS filter）
- **滴血/渗透效果**：CSS 渐变从顶部或边缘向下渗透的暗红色，`background: linear-gradient(to bottom, rgba(139,0,0,0.3), transparent)`
- **呼吸/脉动**：界面元素有极缓慢的明暗脉动（opacity 在 0.85-1.0 之间波动，周期 ≥ 8s）
- **抖动/不稳定**：关键恐怖时刻，文字或界面有微弱的 CSS transform 抖动
- **衰败质感**：面板边缘不规则，有"腐蚀"感的 border 或 clip-path

**恐怖警告弹窗（固定功能，必须实现）**：
- 点击"开始游戏"后，**必须先弹出恐怖警告弹窗**，确认后才进入章节主页
- 弹窗内容：警告文字（如"本游戏包含恐怖内容，可能引起不适。确定继续？"）
- 弹窗 UI 必须本身就很恐怖——不是普通的 confirm 框：
  - 背景极暗，弹窗边缘有不规则腐蚀/裂痕效果
  - 警告文字用恐怖风格（text-shadow + transform 模拟扭曲效果）
  - 文字有轻微抖动或故障闪烁动画
  - "继续"按钮用暗红色，hover 时渗血效果加重
  - "离开"按钮用灰色，点击后回到标题页
- 这个弹窗本身就是恐怖体验的开始

| 选择文字 | 无衬线 | 15-17px | 400 |
| 恐怖强调文字 | 同正文，用 CSS 特效区分 | 同正文 | 400-700 |

> 排版原则：正文行高 ≥ 1.8 倍字号（比其他类型更宽松，给阅读留呼吸感）。段落间距大一些，让黑暗"渗进"文字之间。

### 间距与圆角

- **间距基准**：8px 网格
- **文字区域内边距**：24-32px（比其他类型更宽松）
- **段落间距**：20-28px（刻意留大，让黑暗填充）
- **圆角**：0-4px（方正、硬朗、不友好）

### 必须实现的视觉反馈

| 用户操作 | 视觉反馈要求 |
|---------|------------|
| 进入新章 | 画面渐暗 → 黑屏 → 章标题缓慢抖动显现 |
| 阅读推进 | 文字逐段渐显（fade in），不是瞬间出现 |
| hover 选择 | 按钮边框变红 + 轻微抖动 + 文字微变形 |
| 做出选择 | 墨水扩散/裂痕效果 → 过渡到下一段 |
| 到达恐怖高潮文字 | 文字可能变色/抖动/重影，UI 边缘渗红 |
| 到达结局 | 画面渐变 + CG 缓慢显现 + 结局文字 |
| 屏幕切换 | 渐暗过渡，不能瞬切 |

### 动效标准

- 所有 UI 过渡时长 300-1000ms（比其他类型更慢，营造压抑节奏）
- 使用 ease-in 或 ease-in-out 缓动（慢启动更有压迫感）
- 背景脉动周期 ≥ 10s，极缓慢
- 恐怖特效（抖动/重影/渗血）持续时间短（200-500ms），不能一直抖
- 尊重用户的 `prefers-reduced-motion`（关闭抖动和脉动，保留渐显和过渡）
- 动效服务于恐怖氛围，不是为了炫技

## 十、核心机制约束

> 以下是**不可更改的规则**，开发时必须遵守。

### ⚠️⚠️⚠️ CG 插图生成规则（最高优先级）

> **这是本项目最重要的约束，违反即判定为未完成。**
>
> **CG 插图必须由 AI 生图模型生成，绝对禁止用 CSS / SVG / Canvas / emoji / 文字描述替代。**

**什么是"生图"：**
- ✅ 调用生图模型（如图像生成 API / 工具），输入提示词，获得一张真实的图像文件
- ❌ 用 CSS 渐变、box-shadow、border-radius 画出来的图形 — **这不是生图**
- ❌ 用 SVG path 手动描绘的图形 — **这不是生图**
- ❌ 用 Canvas 代码绘制的图形 — **这不是生图**
- ❌ 用 emoji 或 Unicode 符号拼凑 — **这不是生图**
- ❌ 用纯文字描述"这里应该有一张图" — **这不是生图**

**生图流程（构建时强制执行）：**

1. **为每张 CG 编写详细的生图提示词**：
   - 以 CG 风格提示词基底为前缀
   - 追加该场景的具体描述（场景、氛围、关键元素、光影）
   - 提示词必须足够详细，至少 30 词
2. **调用生图模型生成图片**：
   - 必须实际调用生图工具/API，获得图像数据
   - 如果当前环境有可用的生图工具，必须使用
   - 如果没有可用的生图工具，必须**停下来告知用户**，请求用户提供生图能力，**不可以用 CSS 画一个替代品然后继续**
3. **将图片内嵌到 HTML**：
   - 以 base64 编码嵌入 `<img>` 标签的 src 中
   - 或以 data URL 形式嵌入 CSS background-image 中

**CG 数量要求**：
- 每章至少 1 张场景 CG（关键恐怖场景）
- 结局 1 张 CG
- 背景图 1 张
- 总计：章数 + 1（结局）+ 1（背景）张，全部必须是 AI 生成的真实图片

**如果无法生图怎么办**：
- **不可以**静默地用 CSS 画一个"替代品"然后当作完成了
- **必须**停下来，明确告诉用户："我需要生图能力来生成 CG 插图，但当前没有可用的生图工具。请提供生图工具/API，或者告诉我如何调用。"
- 在生图能力可用之前，可以先完成其他所有部分，但 CG 位置必须留空占位，标注"待生图"

### 叙事规则

**第一人称视角**：
- 全程用"你"作为主语——"你推开门"、"你听到身后有声音"
- 叙事语气从平静逐章变为紧张、慌乱、绝望
- 每段叙事 60-150 字，不要太长——恐怖叙事需要节奏感，短段落 + 留白 = 压迫感

**恐怖递进**：
- 第 1 章：几乎正常，只有 1-2 个极微妙的不对劲（玩家可能注意不到）
- 第 2-3 章：异常越来越明显，但还有合理解释
- 第 4-5 章：细节积累，不安加深，玩家开始主动怀疑
- 倒数第 2 章：合理解释崩塌，恐怖浮出水面
- 最终章：高潮 + 结局

**叙事深度要求**：
- 6-8 章的篇幅意味着故事必须有足够的层次和细节，不能三章讲完核心然后注水
- 每章应有独立的小悬念或揭示，而不只是"继续往前走"
- 人物（如果有）需要有性格、动机、前后变化
- 环境描写要具体、感官化——不是"你感到害怕"，而是描述让人害怕的具体细节
- SKILL 中的知识内容应自然融入叙事，成为故事的血肉而非装饰

**CG 触发时机**：
- CG 应出现在每章最具视觉冲击力的时刻
- CG 显现前有一段铺垫文字，CG 显现后有一段反应文字
- CG 不要太频繁——物以稀为贵，一章 1 张就够

### 选择节点规则

- 每个选择节点恰好 2 个选项
- 两个选项推向不同的下一段叙事（短暂分叉后汇合回主线）
- 选择不计分，不影响结局，纯粹丰富叙事体验
- 选项文字本身要有恐怖叙事感，不能是干巴巴的"A/B"
- 不同选择可以让玩家看到不同的细节/视角，增加重玩价值

### 章末解密小游戏规则

> **每章末尾必须有 1 个解密小游戏，通关才能解锁下一章。** 这是本游戏的核心游戏性来源。

**解密类型库**（开发者为每章选择不同类型，避免重复）：

| 类型 | 玩法 | 实现方式 | 恐怖适配建议 |
|------|------|---------|------------|
| 拼图 | 拖拽碎片拼出完整图片 | CSS grid + 拖拽 | 拼的过程中图片逐渐成形，内容越来越不对 |
| 密码锁 | 输入正确的数字/文字组合 | 输入框 + 验证 | 线索藏在前面的叙事里，像是在撬开不该打开的东西 |
| 滑块解锁 | 滑动多个滑块到正确位置 | range input + SVG | 像是在调频，调到正确频率时听到/看到恐怖内容 |
| 连线匹配 | 把左侧元素和右侧元素正确连线 | SVG line + 拖拽 | 连出来的结果揭示一个恐怖的真相 |
| 翻牌配对 | 记忆翻牌找到配对 | CSS flip + JS | 翻到的图案越来越诡异 |
| 序列还原 | 把打乱的事件/图片排回正确顺序 | 拖拽排序 | 排出来的时间线揭示恐怖真相 |
| 寻找异常 | 在画面中找出不对的地方 | SVG/CSS 画面 + 点击热区 | "这张图有什么不对？"细思极恐 |
| 迷宫 | 操控角色走出迷宫 | Canvas/SVG 绘制 | 在黑暗中摸索，路线暗示某种图案 |

**解密设计约束**：
- 每章必须选择不同的解密类型，**同一个游戏中不可重复使用同一类型**
- 解密的视觉风格必须与恐怖氛围一致（暗色、不安、诡异）
- 解密失败可无限重试，不惩罚
- 累计 3 次提交/确认失败后，出现"查看答案"按钮；点击后显示正确答案，关闭后可继续尝试
- 解密难度适中——不能太简单（30 秒完成没意思），不能太难（卡 10 分钟打断叙事节奏）
- 解密用 CSS / SVG / Canvas / JS 实现（这是交互游戏，不是 CG，可以用代码画）

**每章解密设计表**（填写 spec 时必填）：

| 章 | 解密类型 | 剧情嵌入 | 涉及元素 | 通关条件 |
|----|---------|---------|---------|---------|
| {{CH_NUM}} | {{PUZZLE_TYPE}} | {{NARRATIVE_CONTEXT}}（这个解密在故事里是什么） | {{ELEMENTS}}（几块拼图/几个数字/几条线索） | {{WIN_CONDITION}} |

### 章节主页规则

- 线性解锁：通过第 N 章才能进入第 N+1 章
- 已完成章节可回看（重看叙事和 CG，不重新解密）
- 当前可进入的章节高亮显示
- 未解锁章节灰色锁定
- 进度展示：如 3/6 章

### 结局页

- 最终章解密通关后触发
- 结局 CG 缓慢渐显（2-3s）
- 结局文字在 CG 下方或叠加在 CG 上
- 1 个固定结局
- "重新开始"按钮——重新开始会清除进度
