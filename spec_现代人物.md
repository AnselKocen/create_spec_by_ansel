视觉风格已固定，**不要根据人物特点修改风格方向**。所有现代人物传记统一使用以下视觉基调：

- **视觉基调**：暖白底色 + 羊皮纸质感卡片 + 淡雅柔和色调。无论人物是科技企业家还是文学家，UI 风格保持一致
- **UI 风格关键词**：书卷气、羊皮纸、温润、淡雅、轻盈
- **场景氛围**：所有场景统一使用暖色调羊皮纸卡片，通过文字内容而非 UI 颜色来体现人物特点
- **角色呈现方式**：头像/剪影，风格倾向简洁线条

### 整体布局规则

- **背景图元素集中在右侧**，因此所有内容区域（叙事、对话、决策、章节过渡、章末感悟等）**靠左对齐，占页面左侧 50-60%**，右侧留给背景图
- **所有卡片、气泡、文本容器必须使用羊皮纸风格**：暖白半透明底色（`rgba(255,253,248,0.85)`）+ 极细 1px 暖色边框（`rgba(200,180,150,0.3)`）+ 圆角 8-12px。**禁止使用深色、灰色、黑色底的卡片或气泡**
- 文字颜色统一用深棕（`#2c2c2c`），**禁止白色文字**
- 卡片不使用强阴影，可用极轻的暖色投影（`0 2px 8px rgba(180,160,130,0.15)`）
- 按钮也遵循羊皮纸风格：暖白底 + 暖棕边框，hover 时边框加深，**禁止纯色填充按钮**
- 标题页例外：标题居中显示（因为是入口页，需要视觉焦点居中）

### 动效标准

- 所有过渡时长控制在 200-800ms
- 使用 ease 或 cubic-bezier 缓动曲线，禁止线性运动
- 尊重用户的 `prefers-reduced-motion` 系统偏好
- 动效必须有目的（提供反馈 / 引导注意），禁止为了装饰而加动画

## 十、核心机制约束

> 以下是**不可更改的游戏规则**，CC 开发时必须遵守。

### 历史准确性

- 称谓必须符合当时——注意不同时期的职务、称呼变化
- 人物关系必须真实——谁和谁是什么关系必须基于真实历史，不能编造
- 对话必须符合身份和时代用语——领导、同事、家人、朋友说话语气各不同
- 时间线不能错乱——事件先后顺序、人物年龄与史实一致
- 所有事件必须有真实依据——不能为了剧情效果编造不存在的事件或言论

### 选择不影响剧情

- 故事永远按历史发展，玩家选什么都不改变走向
- 对话选项：2 个选项方向相近，只有语气差异，纯沉浸体验
- 决策选项：3 个选项都"说得通"，选后只看到反馈，不改变后续

### 知识来源

- AI 自身的历史知识（第一优先）+ 用户提供的 skills（第二优先）
- skills 往往只覆盖人物生平的一部分，AI 必须用自己的知识补全完整人生

### 叙事规则

- 全程第二人称「你」
- 现代白话为主，关键处引用人物原话、书信、演讲、日记等原始材料
- 专业术语或外文紧跟括号注释（10 字以内）
- 写具体发生了什么，不写概述
- 事件之间要有过渡

### 对话机制

- speaker 必须是主角以外的人物
- NPC 台词有个性，像真人说话——有情绪、有犹豫、有脾气
- 回应固定 2 个，方向相近，只有语气差异
- followup 不影响走向

### 决策与反馈

- 恰好 3 个选项，`is_authentic: true` 只有 1 个，分值 3/1/0
- 选项文字禁止暗示后果
- **反馈核心原则：故事永远只有一条线——真实历史。反馈的唯一目的是告诉玩家历史上真正发生了什么**
- 与人物相同时：告诉玩家想到一起了，展开选择的经过和后果
- 与人物不同时（必须包含两部分）：① 简短理解玩家（1 句）② 揭示人物真实选择（核心，70% 篇幅）
- **反馈绝对禁止**：描写"玩家选择的后果"、"你错过了""效果远不如"、任何暗示"选错了"的措辞
- 每条 100-180 字，开头和结构各不相同，禁止模板化
- 尽量引用人物原话（用「」包裹）
- 语气温和平等——像朋友聊历史，不是老师判卷

### 章末感悟

- 每章末尾必须有 1 个 ending 场景
- 名言/金句优先选人物自己的原话（演讲、书信、采访、日记、著作等）
- annotation 必填：注明出处和背景（格式：「选自{人物名}{来源类型}《{出处}》，{背景说明}」）
- wisdom（60-100 字）：带感情回望这段人生，让玩家被触动

### 评分系统

- 结局得分 = totalScore / maxScore（百分比）
- 4 档评语：≥85% 知己 / ≥60% 相通 / ≥30% 不同但理解 / <30% 截然不同但不批判
- 3 条结语：按得分匹配名句
- 评语必须包含人物名，引用具体事迹
