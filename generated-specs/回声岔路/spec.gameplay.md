# 《回声岔路》玩法确认稿

> 本文件是 Spec Forge 流程中的 `spec.gameplay.md`，用于先确认玩法。确认后才生成最终 `spec.md`。

## 1. 基础设定

| 维度 | 方案 |
|------|------|
| 产品名称 | 回声岔路 |
| 输出模式 | fixed-content |
| 产品类型 | 分支叙事选择游戏 / 多结局文字冒险 |
| 准确性模式 | fantasy |
| 核心目标 | 玩家通过一次次选择、失败、回看线索，最终抵达好结局 |
| 结局数量 | 7 个：5 个坏结局、1 个好结局、1 个隐藏结局 |
| 主要交互 | 点击推进文本、检查场景线索、做出分支选择、查看结局档案、重开探索 |

| 生图策略 | 需要真实 AI 生图：标题背景、关键章节 CG、每个结局 CG |

## 2. 游戏一句话

玩家在一座只在午夜出现的“零点站”醒来，必须在天亮前找到正确出口。每一次错误选择都会导向坏结局，但坏结局会留下“回声线索”，让玩家在下一轮更接近真正的好结局。

## 3. 世界观与玩家身份

玩家不是英雄，也不是调查员，而是一个忘记自己为何来到零点站的普通人。站内有候车厅、失物招领处、检票口、废弃月台、钟塔和一列永远差一分钟发车的末班车。

零点站的规则不是一开始讲清楚的，而是通过坏结局逐步显形：

- 不能完全服从广播。
- 不能无视陌生人的求助。
- 不能把所有票根都交出去。
- 不能在同一条路上重复三次。
- 看到“自己”时，不一定要逃。

## 4. 核心玩法循环

```text
进入章节场景
→ 阅读当前处境和异常细节
→ 检查 1-3 个可选线索
→ 做出关键选择
→ 触发短期反馈和隐藏变量变化
→ 进入下一场景 / 坏结局 / 好结局
→ 解锁回声线索和结局档案
→ 重开时用已知线索调整路线
```

坏结局不是惩罚，而是学习路径。玩家第一次通关大概率会失败；设计目标是让玩家产生“我刚才哪里选错了，我想再试一次”的冲动。

## 5. 核心机制

### 机制 A：选择改变路线和结局

每章至少 2 个关键选择节点，每个节点提供 2-3 个选项。选项不写成明显的“好/坏”，而是体现价值取向，例如：

- 相信广播 / 相信手写纸条 / 暂时什么都不信
- 赶末班车 / 回候车厅找人 / 去失物招领处
- 交出票根 / 撕下一半 / 藏起票根

选择会影响路线、线索、隐藏变量和结局判定。

### 机制 B：坏结局提供回声线索

每个坏结局都要揭示一个之前不知道的规则。结局页不只显示失败文本，还会解锁一条“回声线索”，加入结局档案。

示例：

| 坏结局 | 玩家学到的回声线索 |
|--------|--------------------|
| 末班误乘 | 广播只会催促你去最短的路，但最短的路不是出口 |
| 票根烧尽 | 票根不能全部交出，至少要保留一部分自我证明 |
| 替身归家 | 逃避“另一个自己”会让它替你离开 |
| 空站留守 | 只救自己会让车站留下一个新的守夜人 |
| 回声成瘾 | 重复同一路线三次会被车站记住并吞没 |

### 机制 C：线索档案与路线回看

游戏提供“结局档案 / 回声档案”，显示已达成结局、已解锁线索、走过的关键选择路径。它不直接告诉玩家正确路线，只把失败中学到的信息清晰整理出来。

回看功能必须鼓励推理，而不是变成攻略按钮。

### 机制 D：隐藏变量

最终实现中可使用 4 个隐藏变量决定结局：

| 变量 | 含义 | 获得方式 |
|------|------|----------|
| `self_proof` | 自我证明，决定是否能抵抗替身 | 保留票根、承认记忆缺口、面对另一个自己 |
| `empathy` | 共情，决定是否能打开真正出口 | 帮助站内滞留者、记住他们的名字 |
| `suspicion` | 警觉，决定是否识破广播和伪线索 | 对矛盾信息进行检查，不盲从 |
| `loop_pressure` | 循环压力，决定车站是否开始主动吞没玩家 | 重复路线、连续忽视线索、强行赶车 |

玩家不直接看到数值，只通过 UI 氛围、文本反馈和档案线索感知状态。

## 6. 章节结构

| 章 | 名称 | 主要场景 | 核心选择 | 可能导向 |
|----|------|----------|----------|----------|
| 序章 | 醒在零点站 | 候车厅长椅、无字时刻表 | 是否相信广播的第一条指令 | 进入主线分支 |
| 第一章 | 候车厅的陌生人 | 候车厅、自动售票机、旧座钟 | 帮不帮一个找不到车票的人 | 共情线 / 赶车线 |
| 第二章 | 失物招领处 | 储物柜、登记簿、破损票根 | 拿回哪一件“属于你”的失物 | 自我证明线 / 替身线 |
| 第三章 | 废弃月台 | 反向行驶的列车、雾中的影子 | 追车、等车、还是走下铁轨 | 多个坏结局入口 |
| 第四章 | 检票口 | 检票员、半张票、手写规则 | 交出票根的方式 | 好结局前置 / 票根坏结局 |
| 第五章 | 钟塔之前 | 站内钟塔、另一个自己、即将天亮 | 面对自己还是让它替你走 | 好结局 / 替身坏结局 / 隐藏结局 |

## 7. 结局设计

| 编号 | 类型 | 名称 | 触发逻辑 | 体验目的 |
|------|------|------|----------|----------|
| E1 | 坏结局 | 末班误乘 | 过度相信广播，持续选择最快赶车路线 | 教玩家“官方声音不可信” |
| E2 | 坏结局 | 票根烧尽 | 在检票口交出完整票根或多次放弃自我证明 | 教玩家保留关键身份线索 |
| E3 | 坏结局 | 替身归家 | 一直逃避另一个自己，最终让替身通过出口 | 教玩家面对核心真相 |
| E4 | 坏结局 | 空站留守 | 全程只追求自保，不帮助任何滞留者 | 教玩家好结局需要共情 |
| E5 | 坏结局 | 回声成瘾 | 重复同一路线过多，忽视新解锁线索 | 防止机械刷路线 |
| E6 | 好结局 | 晨光出口 | 保留半张票根、识破广播、帮助至少两名滞留者、面对自己 | 主目标结局，玩家真正离开车站 |
| E7 | 隐藏结局 | 逆行者 | 已达成至少 3 个坏结局并收集关键回声线索后，在终章选择返回车站深处 | 不是单纯逃离，而是主动打破循环 |

## 8. 好结局路线原则

好结局不能靠第一次乱选碰出来，应由玩家理解规则后主动达成。

必要条件建议：

- 至少解锁过 2 条回声线索，或者在首轮通过隐藏线索也可满足同等条件。
- `self_proof` 达到阈值：保留半张票根，并承认“我忘记了为什么来这里”。
- `empathy` 达到阈值：至少帮助两名站内滞留者。
- `suspicion` 达到阈值：至少两次不盲从广播。
- `loop_pressure` 不能过高：不能连续重复同一条失败路线。

## 9. 隐藏结局路线原则

隐藏结局不是“更好的好结局”，而是给深度玩家的另一种答案。

触发建议：

- 已收集 E1-E4 中至少 3 个坏结局线索。
- 在第五章已经满足好结局条件。
- 面对“另一个自己”时不选择离开，而选择把半张票根交给对方，自己返回候车厅。
- 结局揭示玩家可以成为下一轮中那个留下手写纸条的人，从而让循环中出现真正的缺口。

## 10. 用户动线

```text
标题页
→ 开始游戏
→ 序章
→ 章节选择节点
→ 路线推进
→ 坏结局 / 好结局 / 隐藏结局
→ 结局档案更新
→ 回到标题页或从关键章节重开
→ 逐步逼近好结局
```

关键要求：

- 首次失败后，标题页出现“回声档案”入口。
- 达成任意坏结局后，允许从最近的关键章节重开，不强制每次从头看完序章。
- 重要文本可跳过已读，但首次阅读必须完整呈现。
- 结局档案显示路线，但不直接标注“正确答案”。

## 11. UI 与氛围方向

整体不是纯恐怖，而是“压抑、神秘、带一点温柔的悬疑”。玩家应该感觉自己在一个快要天亮但始终天不亮的地方。

视觉关键词：

- 深夜车站
- 暖黄色旧灯
- 湿冷地面反光
- 纸质票根
- 站内广播的红色提示灯
- 雾、尘埃、细雨、钟摆阴影

UI 设计建议：

- 标题用发光/重影文字，不是普通 h1。
- 选择按钮像旧车票或检票纸条。
- 结局档案像一本被水浸过的站务登记簿。
- 回声线索以票根碎片形式解锁。
- 粒子使用细雨、尘埃、微弱灯尘，不遮挡文本。
- 尊重 `prefers-reduced-motion`，降低动画时改用静态雾层和轻量高亮。

最低资产清单：

| 资产 | 用途 |
|------|------|
| 标题背景图 | 零点站候车厅或月台全局氛围 |
| 章节关键 CG x 5 | 每章一个关键场景 |
| 结局 CG x 7 | 每个结局独立视觉 |

统一画风建议：

```text
dark realistic illustration, cinematic lighting, deep shadow, muted dark tones, lonely midnight train station, atmospheric composition, subtle surreal details
```
