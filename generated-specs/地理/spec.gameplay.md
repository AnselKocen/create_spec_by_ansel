# 地理互动模拟学习游戏玩法确认稿

## 1. 玩法结构名称

**地貌 / 气候参数模拟 + 多题型反馈的地理互动学习游戏**

本产品是一个可填充的地理学习成品框架。后续实现管线会读取随附 `skills/内容文件`，从任意地理相关内容中提取知识点、地理对象、过程、因果、图表和题目材料，填充为一个最终用户可直接学习和游玩的互动产品。

核心体验不是题库，也不是静态地图册，而是：

```text
地理知识点
→ 高精度 SVG / Canvas 地理可视化场景
→ 参数探索
→ 多题型答题
→ 同一场景中的动态反馈演示
→ 章节总结与成就
```

## 2. 输出模式和设计粒度

| 项目 | 确认值 |
|------|--------|
| 项目名称 | 地理 |
| 输出模式 | fixed-content |
| 设计粒度 | 可填充玩法框架 |
| 目标用户 | 多年龄层通用，可由内容文件填充为小学科普、初高中地理、成人通识或综合学习版本 |
| 准确性模式 | strict |
| 技术交付 | 纯前端单文件 HTML + CSS + JS + 内嵌 SVG/Canvas |
| 进度存储 | localStorage |
| 图片缓存 | IndexedDB |

`fixed-content` 表示最终用户打开的是已经填充好的地理互动学习游戏，不需要上传文件、输入内容或操作生成器。内容提取和填充发生在构建阶段。

## 3. 核心循环

每个知识点都必须绑定一个地理可视化场景，并走完四阶段闭环：

1. **讲解阶段**
   - 使用同一个 SVG / Canvas 场景播放动态演示。
   - 讲解文字与图层变化同步，例如讲到“迎风坡降水”时，风向箭头、地形剖面、云量、降水粒子同步变化。
   - 每段讲解 60-120 字，复杂知识点可拆 2-3 段。

2. **探索阶段**
   - 玩家通过滑块、拖拽、开关或地图图层切换调整参数。
   - 可调范围要大，适配地貌、气候、区域、人文地理等不同内容。
   - SVG / Canvas 必须实时响应参数变化，并同步更新数值面板、标注和图例。

3. **答题阶段**
   - 系统锁定某个地理状态或隐藏部分标注。
   - 玩家完成 3 选 1、地图点选/框选、拖拽标注、图层匹配、气候图判读、剖面排序、参数预测等题型。
   - 题目必须基于前两个阶段的同一可视化场景，不得变成脱离图像的纯文字题。

4. **反馈演示阶段**
   - 不管答对答错，都用同一地理场景动态演示答案。
   - 答对：高亮关键图层，播放正确过程，给 30-80 字解释。
   - 答错：先演示错误选择会导致什么，再演示正确结果，给 50-100 字温和纠偏。
   - 答错可重试，但首次作答结果仍用于成就判断。

## 4. 内容文件 / skills 映射

后续管线中的完整内容文件通常位于项目根目录 `skills/` 文件夹。构建本产品时必须读取每个文件全文，而不是只看标题、索引或摘要。

| 内容提取维度 | 映射到的产品模块 |
|--------------|------------------|
| 地理对象：山脉、河流、洋流、城市、区域、气候类型、板块、资源、人口等 | 地图图层、可点击对象、图例项、标注、章节主题 |
| 地理关系：因果、空间邻近、上下游、迎风/背风、海陆位置、板块边界、交通联系等 | 解锁条件、参数响应规则、判断题依据、关系图 |
| 地理过程：风化、侵蚀、沉积、河流改道、季风推进、城市扩张、板块运动等 | 动态 SVG / Canvas 演化动画、阶段状态、反馈演示 |
| 地理规则：气候成因、经纬度规律、海拔递减、洋流影响、人口分布规律等 | 可调参数、验证规则、题目答案、自检项 |
| 地理数据：气温、降水、海拔、坡度、流速、人口密度、资源储量等 | 数值面板、图表、气候图、热力层、参数范围 |
| 可验证事实：地名、区域特征、真实案例、图表结论、教材定义 | strict 准确性验收、题目正确答案、解析依据 |
| 视觉主题：地图风格、地貌类型、区域气质、自然环境 | UI 氛围、背景图 prompt、粒子类型、过渡动效 |

## 5. 章节和内容规模

默认规模沿用物理 / 化学互动学习规格：

- 5-7 章。
- 每章 3-4 个知识点。
- 每个知识点 1 道主问题，复杂知识点可增加 1 道进阶题。
- 全书总题量约 20-25 题。
- 每章末尾 1 个知识总结。
- 每章 3 个成就：全对成就、全错调侃成就、探索创意成就。

章节可由内容文件自动映射为：

```text
自然地理基础 → 地貌过程 → 气候系统 → 水文与生态 → 区域综合 → 人文地理 / 城市与资源
```

也可以由具体内容文件决定，例如中国地理、世界地理、区域研学、地貌专题、气候专题等。

## 6. 参数范围和交互变量

本框架需要支持较大范围的动态参数调整，实际参数由 `skills/内容文件` 按知识点填充。

推荐参数池：

| 参数类型 | 示例参数 | 可视化响应 |
|----------|----------|------------|
| 位置参数 | 纬度、经度、海陆位置、距海距离 | 日照角度、气候带、地图定位、海陆热力差异 |
| 地形参数 | 海拔、坡度、坡向、地形起伏、山脉高度 | 等高线密度、剖面抬升、阴影、河流流向 |
| 气候参数 | 气温、降水、湿度、气压、风向、风速、季节 | 风场箭头、雨带粒子、云量、气候图曲线 |
| 水文参数 | 河流流速、流量、含沙量、侵蚀强度、沉积速率 | 河道摆动、侵蚀/堆积区域、三角洲推进 |
| 海洋参数 | 洋流方向、寒暖流、海温、上升流强度 | 洋流线、海温色带、沿岸降水变化 |
| 地质参数 | 板块运动方向、碰撞速度、断层活动、火山活动 | 板块箭头、褶皱隆起、地震波、火山喷发提示 |
| 人文参数 | 人口密度、城市化程度、交通可达性、资源分布 | 热力图、城市扩张、流线、区位选择反馈 |

交互控件包括滑块、双向滑块、地图拖拽点、图层开关、时间轴、季节切换、剖面拖拽线、风向旋钮、参数重置和播放/暂停。

## 7. 题型设计

题型要服务地理图像判读和过程理解，默认包含：

| 题型 | 玩法 | 必须依赖的可视化 |
|------|------|------------------|
| 3 选 1 | 根据场景判断正确结论 | 当前地图 / 剖面 / 气候图状态 |
| 地图点选 | 在地图上点击正确区域、城市、河段、气候带或地貌单元 | SVG 地图热区 |
| 地图框选 | 框出受某因素影响最大的区域 | SVG 地图 + 半透明选择框 |
| 拖拽标注 | 把地理名词、因果标签、风向、洋流名拖到正确位置 | SVG 标注锚点 |
| 图层匹配 | 将现象图层与成因图层匹配，例如降水分布 ↔ 地形迎风坡 | 多图层地图 |
| 气候图判读 | 根据气温 / 降水柱线图判断气候类型或成因 | SVG 气候图 |
| 剖面排序 | 排列地貌演化阶段或地形剖面顺序 | SVG 剖面卡片 |
| 参数预测 | 调参前预测结果，再播放反馈验证 | 参数面板 + 动态模拟 |

所有题型必须提供可视化反馈，不允许只显示“正确 / 错误”。

## 8. 高精度 SVG / Canvas 可视化方向

SVG 是本产品的核心资产，不是装饰。最终 spec 需要把 SVG 要求写到高优先级约束中。

### SVG 必须承担的内容

- 分层设色地图：高程、温度、降水、人口密度、资源分布等。
- 等高线与地形阴影：等高线密度、标高点、山脊/谷地识别。
- 地形剖面：地表曲线、海拔标尺、岩层/植被/积雪/河谷剖面。
- 风场与洋流：箭头方向、线宽/颜色表示强度，流线可动画。
- 气候图：气温折线、降水柱状、季节刻度、气候类型标注。
- 板块与构造图：板块边界、运动箭头、碰撞/张裂/俯冲剖面。
- 水文过程图：河流流向、侵蚀岸/堆积岸、三角洲、洪泛区。
- 人文地理图层：人口热力、城市节点、交通流线、资源区位。

### SVG 质量要求

- 每个知识点至少有一个 SVG 或 SVG + Canvas 混合的核心可视化。
- 同一知识点四阶段必须共用同一个可视化，通过 `vis_state` 切换状态。
- 地图图层必须有清晰图例、比例/方向提示、坐标或区域标注。
- 线条、填充、阴影、图例、标注要分层管理，避免混成一张不可维护的图。
- 关键地理量必须可视编码：颜色、线宽、透明度、箭头长度、粒子密度、等值线密度等都要有对应含义。
- 地理标注要有避让规则，不能遮挡关键图形；动态标注需要平滑移动。
- 可交互对象要有 hover / active / selected 状态，且反馈不能影响地理判断。
- 移动端 SVG 必须可缩放、可平移或重排，核心标注不能被裁切。

### Canvas 适用范围

Canvas 用于大量粒子或连续动态过程：

- 降水粒子、雪线变化、风沙移动。
- 河流泥沙扩散、海浪/洋流粒子、云团移动。
- 火山灰、地震波、城市扩张热力渐变。
- 大量风场/流场粒子的逐帧动画。

SVG 与 Canvas 可以混用：SVG 负责精确地图、剖面、标注、坐标、图例；Canvas 负责连续粒子和场变化。

## 9. 页面 / 状态结构

```text
首次打开
└── 图片缓存检查
    ├── cache hit：载入背景图 → 标题页
    └── cache miss：资产准备页 → 生成背景图 → 写入 IndexedDB → 标题页

标题页
├── 开始学习
└── 继续学习

章节主页
├── 已完成章节：可回看
├── 当前章节：高亮可进入
└── 未解锁章节：锁定

章节学习
├── 章节过渡
├── 知识点闭环 × 3-4
│   ├── 讲解阶段
│   ├── 探索阶段
│   ├── 答题阶段
│   └── 反馈演示阶段
└── 章末总结

通关页
├── 成就墙
├── 各章一句话回顾
└── 重新开始 / 回看章节
```

## 10. 生图资产范围和时机

本产品需要真实 AI 生图，但仅用于标题页 / 全局背景图，核心地理图、地图、剖面、气候图和知识可视化必须由 SVG / Canvas / HTML 实现，不能用位图替代。

| 项目 | 确认值 |
|------|--------|
| 是否需要真实位图生图 | 需要 |
| 生图资产范围 | 标题页 / 全局背景图 |
| 核心地理可视化 | SVG / Canvas，不使用 AI 图片替代 |
| IMAGE_GENERATION_TIMING | first-run-cache |
| 缓存方式 | 首次打开生成缺失背景图，写入 IndexedDB，之后只读缓存 |
| build-time 例外 | 无 |

背景图方向：

```text
immersive futuristic geography observatory, dynamic terrain hologram, contour maps, atmospheric data panels, climate visualization screens, deep cinematic lighting, professional cartography lab, no people, no readable text, wide background composition
```

最终 spec 需要保留完整生图契约：`IMAGE_ASSET_MANIFEST`、`IMAGE_ASSET_RUNTIME_STATE`、IndexedDB Blob 缓存记录、prompt hash、cache key、生成锁、失败重试、cache hit 不调用生图 API。

## 11. 视觉方向

视觉气质：**沉浸酷炫的专业地理观测站 + 精密地图册 + 动态地形沙盘**。

关键词：

- 深色或冷调地图仪表面板。
- 等高线、经纬网、地形阴影、透明图层叠加。
- 风场箭头、洋流线、雨带粒子、云层流动。
- 动态剖面、参数仪表、图例面板、数据读数。
- 章节过渡可使用地图扫描、地形抬升、等高线展开、卫星轨道扫过等效果。
- 配色不能只停留在深蓝科技风；UI 底色保持克制，地图、地貌、气候、水文、灾害、人文图层要使用更丰富的地理语义色板。

UI 需要酷炫，但不能牺牲图表判读：

- 文字对比度必须达标。
- 地理编码色彩不能为了风格而混淆含义。
- 同一核心可视化最多突出 2-3 个强色系，其余图层降饱和或提高透明度，避免画面变花。
- 题目状态和探索状态要视觉区分。
- `prefers-reduced-motion` 下减少粒子和过渡，但保留核心可视化演示。

## 12. 准确性模式

准确性模式为 `strict`。

必须严格处理：

- 地名、区域事实、经纬度/海陆位置关系。
- 气候类型、气温降水规律、季风/洋流/地形雨等因果。
- 地貌成因、板块运动、流水侵蚀沉积、风化风蚀等过程。
- 人口、城市、资源、交通等人文地理判断。
- 图表数据、题目答案、错误选项、反馈解析。

允许创作发挥的范围：

- UI 氛围。
- 成就名称和描述。
- 鼓励语、章节过渡文案。
- 非事实性的任务包装。
- 背景图美术风格。

不得为了酷炫视觉改错地理规律，不得凭空编造核心知识、真实区域事实或题目答案。

## 13. 待确认点

当前玩法方案已根据用户确认写入以下默认值：

- 输出模式：`fixed-content`。
- 项目名：`地理`。
- 结构：地貌 / 气候参数模拟 + 多题型答题。
- 内容范围：任何地理相关内容，通用框架。
- 目标用户：多年龄层通用，通过内容难度填充区分。
- 内容规模：5-7 章，每章 3-4 个知识点，总题量约 20-25。
- 生图：仅标题页 / 全局背景图，`first-run-cache`。
- 核心可视化：极高要求 SVG / Canvas。
- 准确性：`strict`。
- 技术交付：纯前端单文件，localStorage + IndexedDB。

请用户确认：是否按本玩法方案继续生成最终 `generated-specs/地理/spec.md`。
