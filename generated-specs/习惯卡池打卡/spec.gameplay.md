# 习惯卡池打卡 - 玩法确认稿

## 1. 玩法结构名称

**抽卡式习惯养成打卡产品**

本产品是一个 `fixed-content` 成品框架：最终用户打开后直接体验已经填充好的习惯卡池、抽卡、打卡、日历记录和奖励系统；不会让最终用户上传内容或操作生成器。

后续管线会读取随附 `skills/内容文件`，从一堆不一定专门讲“习惯养成”的内容里抽象出可执行的 40-50 条通用习惯卡。不同输入内容可以生成不同主题、不同类别、不同画风和不同奖励体系，但最终体验结构保持一致。

## 2. 输出模式与设计粒度

| 项目 | 设定 |
|------|------|
| 输出模式 | `fixed-content` |
| 设计粒度 | 可填充玩法框架 |
| 项目目录 | `generated-specs/习惯卡池打卡/` |
| 玩法核心 | 从内容文件提炼习惯卡池，用户每日抽卡并打卡，用日历和奖励强化持续行动 |
| 内容规模 | 40-50 张习惯卡，若内容不足可生成 24-36 张基础卡；若内容丰富可扩展到 60 张以内 |
| 准确性模式 | `semi-strict` |

## 3. 核心循环

```text
打开产品 → 查看今日状态 → 抽取今日习惯卡 → 选择接受/换抽有限次数 → 完成行动 → 打卡并记录心情/备注 → 获得奖励 → 日历更新连续天数和完成率 → 次日回访
```

核心体验不是“管理任务清单”，而是“从内容中抽到今日行动”。抽卡带来轻微期待感，日历负责长期记录，奖励系统负责让用户感觉每次完成都有回响。

## 4. 内容如何进入玩法

### 内容抽象原则

随附内容文件可能是自我管理、健康、学习、情绪、阅读、工作方法、人物经验、生活方式或更抽象的知识内容。实现者需要从中提炼可执行行为，而不是只摘录原文。

| 内容提取维度 | 映射到产品模块 |
|--------------|----------------|
| 反复出现的建议、原则、价值观 | 习惯卡名称与坚持理由 |
| 可执行动作、步骤、练习、提醒 | 每日最小行动 |
| 场景、对象、问题、关系 | 适用场景与卡片标签 |
| 风险、禁忌、边界、注意点 | 卡片注意事项与温和提醒 |
| 情绪、节奏、审美、隐喻 | 奖励图片主题、UI 氛围、粒子动效 |
| 内容结构或章节差异 | 习惯分类、卡池分组、奖励系列 |

### 必须来自内容文件的部分

- 习惯卡名称。
- 习惯卡的每日行动建议。
- 习惯分类可以来自内容文件，也可以由实现者根据内容聚类生成。

### 可以创作发挥的部分

- 奖励名称、成就文案、稀有度包装。
- 打卡反馈语、鼓励语、视觉意象。
- UI 主题和奖励画风，但必须能说明与内容文件的气质有关。

### 通用性约束

习惯卡应适合普通用户日常使用，避免假设用户有专业背景、特殊设备、固定职业、特定年龄或强医学需求。若内容文件本身不适合直接转成习惯，允许抽象为“观察、记录、反思、尝试、整理、复盘、沟通、休息、学习、微行动”等通用行为。

## 5. 习惯卡槽位

每张习惯卡建议包含以下字段：

| 字段 | 说明 |
|------|------|
| `habit_id` | 稳定 ID |
| `habit_name` | 从内容文件提炼出的习惯名 |
| `category` | 内容聚类生成的习惯类别 |
| `daily_micro_action` | 今日可完成的最小行动，必须具体、低门槛 |
| `why_it_matters` | 简短说明这条习惯和内容文件核心观点的关系 |
| `estimated_time` | 1 分钟、3 分钟、5 分钟、10 分钟、15 分钟等 |
| `difficulty` | `easy` / `normal` / `challenge` |
| `rarity` | `common` / `rare` / `epic`，用于抽卡反馈，不代表价值高低 |
| `best_moment` | 适合早晨、午间、睡前、通勤、工作间隙、任意时间等 |
| `checkin_prompt` | 打卡时的提示问题 |
| `source_trace` | 可追溯到内容文件的原文位置、章节或摘录摘要 |

习惯卡不能写成宏大目标，例如“保持健康生活”。必须转成当天能执行的小动作，例如“睡前写下明天最重要的一件事”。

## 6. 抽卡机制

### 每日抽卡

- 每天默认抽 1 张主卡。
- 用户可以选择额外抽 1-2 张加练卡，但不能用强刺激诱导过度打卡。
- 每日允许有限换抽，例如 1-3 次，用于跳过当天不适合的习惯。
- 换抽不惩罚用户，但界面应鼓励选择“今天真的能做的一张”。

### 稀有度

稀有度用于仪式感，不用于制造焦虑：

| 稀有度 | 含义 |
|--------|------|
| `common` | 低门槛、适合每天做的基础习惯 |
| `rare` | 稍有挑战、需要一点准备或专注 |
| `epic` | 特殊主题、适合周末或高能量日完成 |

### 卡池平衡

- `common` 建议占 60%-70%。
- `rare` 建议占 20%-30%。
- `epic` 建议占 5%-10%。
- 同一类别不应连续多日过度重复，避免体验单调。

## 7. 打卡与日历记录

用户完成习惯后进入打卡面板，至少记录：

| 记录项 | 说明 |
|--------|------|
| 完成习惯 | 当天完成的习惯卡 ID，可多张 |
| 心情 | 以 3-5 档情绪或主题化表情按钮记录 |
| 备注 | 用户可写一句当天感受 |
| 连续天数 | 自动计算 streak |
| 完成率 | 按 7 日、30 日、全部时间统计 |
| 奖励记录 | 当天获得的图片、徽章、卡背或称号 |

日历需要支持：

- 月视图查看每天完成状态。
- 点击日期查看当天习惯、心情、备注和奖励。
- 连续打卡天数高亮。
- 断签不羞辱用户，可提供“重新开始今日一张”的温和回归入口。

## 8. 奖励系统

采用混合奖励：

| 奖励类型 | 触发方式 | 说明 |
|----------|----------|------|
| 即时奖励 | 每次完成打卡 | 翻卡动画、粒子、鼓励语、少量积分或能量 |
| 图片奖励 | 完成特定卡、类别、连续天数或里程碑 | 需要真实 AI 生图 |
| 成就徽章 | 连续 3/7/14/30 天，完成某类别若干次 | 可用 SVG/CSS 徽章或真实图像奖励混合 |
| 卡背/主题 | 完成某类习惯或稀有卡 | 改变抽卡视觉 |
| 收藏图册 | 汇总已解锁奖励图 | 让奖励可回看、可收藏 |

奖励不应宣称治疗、变瘦、变强、治愈焦虑等确定效果。它表达的是“你完成了一个小行动”，不是健康结果保证。

## 9. 页面 / 状态粗结构

```text
首次打开
→ 图片缓存检查 / 视觉资产准备页
→ 标题页
→ 今日抽卡页
→ 习惯卡详情页
→ 打卡记录页
→ 奖励揭示页
→ 日历页 / 奖励图册 / 卡池图鉴
```

### 关键页面

| 页面 | 职责 |
|------|------|
| 资产准备页 | `first-run-cache` 生图策略下检查缓存并生成缺失图片，写入 IndexedDB |
| 标题页 | 展示产品名、今日连续天数、进入抽卡 |
| 今日抽卡页 | 执行抽卡、换抽、接受今日卡 |
| 习惯卡详情页 | 展示行动建议、预计耗时、坚持理由、注意事项 |
| 打卡页 | 记录完成、心情、备注 |
| 奖励揭示页 | 展示即时奖励、图片奖励或成就 |
| 日历页 | 查看每日记录、连续天数、完成率 |
| 卡池图鉴 | 查看已抽到/已完成/未解锁习惯卡 |
| 奖励图册 | 查看真实生图奖励、徽章、卡背和里程碑 |

## 10. 生图资产范围与时机

本产品需要真实 AI 生图，策略为 `first-run-cache`。

### `IMAGE_GENERATION_TIMING`

`first-run-cache`

### first-run-cache 资产

| 资产 | 数量建议 | 用途 |
|------|----------|------|
| 标题页背景 | 1 | 首屏氛围 |
| 默认卡背 | 1-3 | 抽卡时的基础视觉 |
| 默认奖励封面 | 1 | 奖励图册空状态或封面 |
| 类别奖励图 | 每个习惯类别 1 张 | 完成类别进度时解锁 |
| 连续打卡里程碑图 | 3/7/14/30 天等 | streak 奖励 |
| 稀有习惯奖励图 | 若干张 | 完成 rare/epic 习惯后解锁 |
| 特殊结算图 | 1-3 张 | 30 日回顾、月度总结 |

首次打开 app 时先检查 IndexedDB。缓存命中时直接读取 Blob、load/decode 并进入标题页；缓存缺失时显示主题化资产准备页，逐张调用真实生图并写入 IndexedDB。之后刷新或下次打开，只要 `cache_key` 命中且图片可正常加载，就不得再次调用生图工具/API。

所有真实图片必须写入静态 `IMAGE_ASSET_MANIFEST`，包含 asset id、用途、统一画风基底、完整 prompt、negative prompt、比例、生图时机、required、plan_status、initial_runtime_status、cache key、prompt hash、seed source 和 storage driver。运行时必须维护 `IMAGE_ASSET_RUNTIME_STATE`，用 ready 状态决定 required 图片是否能放行核心体验。

如果当前实现环境没有可用生图能力，后续实现必须停下来说明缺少生图工具，不能用 CSS/SVG/Canvas/emoji/文字占位冒充真实生图。

## 11. 动态画风选择方法

最终 spec 不固定单一画风，而是要求实现者根据输入内容动态选择 `STYLE_PROMPT_BASE`。可从以下维度组合：

| 维度 | 可选方向 |
|------|----------|
| 媒介 | 水彩、油画、版画、彩铅、拼贴、3D、像素、矢量、摄影、纸雕、胶片 |
| 情绪 | 清爽、温柔、坚定、安静、热烈、神秘、复古、未来感、治愈、专注 |
| 内容来源气质 | 生活方式、健康管理、职场效率、情绪整理、学习成长、哲学思辨、自然观察、人物经验 |
| 材质 | 手账纸、玻璃、金属、木纹、织物、便签、胶片颗粒、发光晶体、陶瓷 |
| 构图 | 卡片头像式、宽幅场景、贴纸组合、图鉴版式、日历插画、徽章中心构图 |
| UI 质感 | 生活手账、精致卡牌、轻游戏 HUD、清爽仪表盘、收藏图册、玻璃拟态、复古票券 |
| 动效气质 | 纸张翻动、星尘、光点、贴纸弹跳、墨迹扩散、进度环发光、卡牌翻转 |

### 画风候选库

后续实现可根据内容文件动态选择或混合以下方向：

| 画风方向 | 适合内容 |
|----------|----------|
| 清爽生活手账风 | 普通生活习惯、个人成长、轻量自我管理 |
| 水彩晨间日记风 | 情绪照顾、慢生活、阅读和反思 |
| 复古票券收藏风 | 打卡、里程碑、旅行式成长路径 |
| 玻璃拟态健康仪表盘 | 健康记录、睡眠、饮水、运动、节律 |
| 扁平矢量信息图 | 效率方法、学习计划、工作习惯 |
| 纸雕贴纸拼贴风 | 轻松可爱、亲和、低压力习惯 |
| 新艺术植物装饰风 | 自然、身体感受、长期生长隐喻 |
| 像素任务卡风 | 游戏化更强、每日任务、成就收集 |
| 复古未来个人终端 | 科技效率、数据记录、专注系统 |
| 胶片摄影生活风 | 真实生活、城市日常、个人观察 |
| 柔和 3D 小物风 | 轻奖励、可收藏物件、卡池抽奖感 |
| 极简禅意线条风 | 冥想、整理、呼吸、低刺激产品 |
| 国潮手账风 | 中文传统内容、节气、身体调养、文化主题 |
| 博物志图鉴风 | 从自然、知识、观察类内容抽象习惯 |
| 梦幻星图风 | 愿望、长期计划、成长路径、里程碑 |

### 选择规则

- 如果内容偏健康、自律、睡眠、运动：优先清爽、玻璃拟态、生活手账。
- 如果内容偏情绪、反思、阅读：优先水彩、胶片、极简线条。
- 如果内容偏效率、学习、工作流：优先扁平矢量、复古未来、信息图。
- 如果内容偏文化、自然、哲学：优先国潮、博物志、新艺术、禅意线条。
- 若内容混合，允许为不同习惯类别分配同一基底下的子风格，但所有图片仍必须共享统一主风格基底，避免奖励图册割裂。

## 12. UI 与动效方向

- UI 应兼具卡池游戏的期待感和习惯工具的清爽感。
- 首页、抽卡、奖励揭示要有仪式感；日历和统计要清晰、克制、可长期使用。
- 卡片翻转、稀有度光效、完成粒子、日历点亮需要主题化设计。
- 粒子建议根据画风动态选择：纸屑、星尘、光点、花瓣、便签碎片、像素粒子、胶片尘埃等。
- 尊重 `prefers-reduced-motion`，减少或关闭大幅翻转、粒子和背景漂移。

## 14. 准确性模式

`semi-strict`

要求：

- 自我管理和健康建议必须合理、温和、低风险。
- 不承诺医学、心理治疗、减肥、疾病改善等结果。
- 不要求用户执行高风险行为、极端节食、过度运动、睡眠剥夺或专业医疗替代动作。
- 若内容文件不适合转成健康习惯，应抽象为观察、记录、反思、整理、学习、沟通、休息等通用习惯。
- 游戏化文案、奖励名、视觉隐喻可以自由创作，但不能扭曲习惯行动本身。
