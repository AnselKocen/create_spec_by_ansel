## 四、视觉风格规范

> 本节强制约束游戏的视觉表现。即便完全使用 HTML + CSS + JS，也必须达到以下标准。

### 视觉风格（由人物和时代决定）

> **UI 风格不是固定的，必须根据人物的时代背景和性格气质来设计。** 开发者应先分析人物是谁、什么时代、什么气质，再决定整体视觉方向。

**风格确定流程（强制）**：
1. 分析人物的时代背景（古代/近代/现代？中国/西方/其他？）
2. 分析人物的性格气质（豪放/内敛？文人/武将？悲壮/潇洒？）
3. 确定视觉风格关键词和主色系
4. 所有 UI 元素围绕这个风格设计

**配色由开发者根据人物和时代自行决定，遵循以下原则**：
- 60-30-10 法则：60% 主色、30% 辅助色、10% 强调色
- 主色系来自人物的时代和气质
- 文本与背景对比度 ≥ 4.5:1

**排版原则**：
- 全站不超过 2 种字体
- 正文行高 ≥ 1.6 倍字号
- 正文区域最大宽度 800px 居中
- 间距遵循 8px 网格

### CG 场景插图生成（必须实现）

> **每个关键场景必须有 AI 生成的 CG 插图。** 这是传记游戏的视觉核心——让玩家"看见"人物的人生。

**什么是"生图"：**
- ✅ 调用生图模型，输入提示词，获得一张真实的图像文件
- ❌ 用 CSS/SVG/Canvas 画的图形 — 不是生图
- ❌ 用 emoji 或文字描述 — 不是生图

**CG 风格提示词基底**：
- 开发者根据人物时代和视觉风格，确定一组 CG 风格提示词基底
- 所有 CG 共享这组基底，保证画风统一
- 每张 CG 追加具体的场景描述

**必须生成的图片**：
- **首页背景图** 1 张 — 体现人物和时代的氛围，作为最底层图层，不遮挡 UI
- **每章至少 1 张场景 CG** — 在章节的关键场景/情感高潮处展示
- **每个结局 1 张结局 CG** — 3 个结局各 1 张
- **章节过渡可选** — 如果有合适的场景也可以生图

**CG 展示方式**：
- CG 出现在叙事段落中，渐显展示
- 文字叠加在 CG 下方或上方（半透明底色保证可读性）
- CG 是氛围强化，不是装饰——每张 CG 对应一个具体的叙事场景

**如果无法生图**：
- 必须停下来告知用户，不可以用 CSS 画替代品冒充

### 必须实现的视觉反馈

| 用户操作 | 视觉反馈要求 |
|---------|------------|
| 进入章节 | 章节过渡屏动画（章名 + 年代），不能瞬切 |
| 阅读叙事 | 文字逐字或逐段显现动画 |
| 对话选择 | 选项 hover 高亮，选中后玩家回复显现 + NPC 后续显现 |
| 做出决策 | 选项 hover 高亮，选中后跳转反馈页 |
| 决策反馈 | 反馈文字动画显现 |
| 成就解锁 | toast 通知动画（滑入 + 自动消失） |
| 章末感悟 | 诗句逐行显现，注释和感悟随后淡入 |
| 屏幕切换 | 淡入淡出过渡，不能瞬切 |
| 悬停可交互元素 | 明确的 hover 状态反馈 |
| 空状态 | 无（线性游戏，不存在空状态） |

### 动效标准

- 所有过渡时长控制在 200-800ms
- 使用 ease 或 cubic-bezier 缓动曲线，禁止线性运动
- 尊重用户的 `prefers-reduced-motion` 系统偏好
- 动效必须有目的（提供反馈 / 引导注意），禁止为了装饰而加动画

### 核心路径

```
标题页 → 开始游戏 → 共享章节（章1-2）→ 关键分支决策 → 支线章节 → 关键分支决策 → 支线章节 → 结局页
                                                ↘ 死胡同 → 回退到分支点重选

| 源功能 | 流向 | 目标功能 | 流转方式 |

| 章节过渡 | → | 场景播放 | 点击"进入" |

### 导航结构

```
标题页
├── 开始游戏 → 进入共享段第 1 章
└── 继续游戏 → 恢复存档进度（仅有存档时显示）

游戏中（分支推进）
├── 章节过渡屏
├── 叙事场景
├── 对话场景
├── 普通决策 → 反馈场景
├── 关键分支决策 → 进入对应支线章节
├── 死胡同场景 → "回到上一个选择"按钮 → 回到分支点

结局页（3 个不同结局，由路线决定）
├── 结局叙事 + 诗句 + 感悟
├── 路线回顾 + 成就徽章
└── 重新开始 → 返回标题页（探索其他分支）

> 本游戏有分支结构。共享段线性推进，分支点后进入不同支线。死胡同可回退到上一个分支点，不需要重来。

## 九、交互设计

### 交互模式选择

**本项目选择**：新手友好

**选择原因**：目标用户是历史文化爱好者而非硬核玩家，交互应极简——只需要「点击推进」和「选择选项」两种操作。

### 操作映射

| 操作 | 实现方案 | 说明 |
|------|---------|------|
| 推进叙事 | 点击"继续"按钮 | 文字先动画显现，按钮在文字完毕后出现 |
| 对话选择 | 点击 2 个回应按钮之一 | 选后显示玩家气泡 + NPC 后续，再显示继续按钮 |
| 普通决策 | 点击选项按钮之一 | 选后跳转反馈屏，不可撤回 |
| 关键分支 | 点击选项按钮之一 | 选后直接进入对应支线章节或死胡同 |
| 死胡同回退 | 点击"回到上一个选择"按钮 | 回到最近的关键分支点 |
| 章末继续 | 点击继续按钮（延迟出现） | 按钮在诗句/感悟全部显现后出现 |
| 存档 | 自动 | 每次状态变化自动写入 localStorage（含分支路径记录） |
| 读档 | 点击标题页"继续游戏" | 从存档恢复到对应章节和场景 |

### 交互反馈

| 操作 | 视觉反馈 | 状态变化 |
|------|---------|---------|
| 阅读叙事 | 文字动画显现 | currentScene +1 |
| CG 场景 | CG 插图渐显 + 叙事文字 | currentScene +1 |
| 选择对话回应 | 回应按钮消失，玩家气泡显现，NPC 后续显现 | currentScene +1 |
| 普通决策 | 屏幕切换到反馈页 | choices 记录 |

| 进入死胡同 | 死胡同叙事渐显 + 回退按钮 | 无永久状态变化 |
| 点击回退 | 过渡动画 → 回到分支点 | 恢复到分支点状态 |
| 解锁成就 | toast 通知滑入后自动消失 | achievements 数组新增 |
| 进入新章 | 章节过渡屏 | currentChapter 更新, currentScene = 0 |
| 到达结局 | 过渡动画 → 结局 CG 渐显 → 结局叙事 | finished = true |

## 十、核心机制约束

> 以下是**不可更改的游戏规则**，CC 开发时必须遵守。

### 历史准确性

- 称谓必须符合当时——不能用后世追封的名号
- 人物关系必须真实——谁和谁是什么关系必须基于真实历史
- 对话必须符合身份——皇帝、大臣、家人说话语气各不同
- 时间线不能错乱——事件先后顺序、人物年龄与史实一致

### 分支系统

> **选择会影响故事走向。** 游戏有 2-3 个关键分支点，不同选择走向不同的支线，最终通向 3 个不同的结局。

**两种选择类型**：

- **普通对话选择**：2 个选项方向相近，只有语气差异，纯沉浸体验，不影响走向
- **关键分支决策**：2-3 个选项，直接决定后续走哪条支线。选错可能走进死胡同

**分支地图（填 spec 时必须画出）**：

```
{{BRANCH_MAP}}
```

> 分支地图模板（填写时替换）：

```
第一幕：共享段（所有玩家看到相同内容）
  章1（少年/早期）→ 章2（成长/转折前奏）
                        ↓
                   ══ 分支点1 ══
                  /       |       \
               选项A    选项B    选项C
                /         |         \
  第二幕：   支线甲     支线乙     死胡同→回退
           章3A        章3B
           （走历史    （走另一条
            真实路线）   合理路线）
              ↓           ↓
         ══ 分支点2A ══  ══ 分支点2B ══
          /     \          /     \
       选项A  选项B     选项A   选项B
        /       \        /        \
  第三幕：
     章5A     死胡同    章5B      死胡同
     （继续    →回退   （继续     →回退
      深入）            展开）
       ↓                 ↓
     结局1              结局2              结局3
   （历史真实结局）  （合理假设结局）   （另一种可能）
```

**分支设计原则**：

1. **共享段占 30-40%**：前 2 章所有玩家看到相同内容，建立人物基础和世界观
2. **分支段占 50-60%**：不同支线有不同的事件、对话、场景
3. **结局段占 10-20%**：3 个结局各 1-2 个场景
4. **支线之间可以共享部分场景**：同一个历史事件可以从不同角度叙述
5. **死胡同要短**：2-3 段叙事说明"这条路走不通了"，然后提供回退按钮
6. **跟历史不同不等于"错"**：有些分支选了和主角不同的路反而可能更好，只是走向了不同的故事

**回退机制**：

- 走进死胡同时，显示一段简短的叙事（为什么这条路走不通）
- 提供"回到上一个选择"按钮，回退到**最近的关键分支点**，不是重来整个游戏
- 回退时保留已读的叙事记忆（玩家知道那条路走不通了）
- localStorage 记录每个分支点的选择历史

**结局设计框架**：

| 结局 | 调性 | 内容方向 |
|------|------|---------|
| 结局1 | 历史真实结局 | 走最接近真实历史的路线，看到人物真正的命运和选择 |
| 结局2 | 合理假设结局 | 基于人物性格和时代背景，如果在某个关键点做了不同选择，可能会怎样 |
| 结局3 | 另一种可能 | 最远离历史的路线，但仍然合理——展示"如果一切都不同" |

> 3 个结局都不批判，都有各自的感悟和意义。不存在"好结局"和"坏结局"，只有"不同的人生"。

### 知识来源

- AI 自身的历史知识（第一优先）+ 用户提供的 skills（第二优先）
- skills 往往只覆盖人物生平的一部分，AI 必须用自己的知识补全完整人生

### 叙事规则

- 全程第二人称「你」
- 文白混搭——日常白话亲切易读，关键处引用原文
- 生僻词紧跟括号注释（10 字以内）
- 每段 100-200 字，要有**具体的人名、地名、动作、感官细节**
- 写具体发生了什么，不写概述。反例："你经历了很多"；正例："大雪封路，你裹着单衣在驿站等了三天"
- 不要只写大事件——也要写**日常**：人物的生活细节、心理活动、与身边人的相处
- 要写出**感官**：天气、声音、气味、场景的画面感
- 事件之间要有**自然过渡**：不要从一个大事件直接跳到另一个，要写出期间的日常、心理变化、氛围转换
- **章内场景必须按时间顺序推进，先发生的事先讲，后发生的事后讲，禁止倒叙或时间跳跃**
- **每章的叙事围绕一件事循序渐进：先交代前因和处境，再通过对话展开事件发展，最后推向决策和章末高潮。禁止开章直接进入关键冲突，要让玩家先知道"事情是怎么走到这一步的"**
- 支持 `\n` 换行分段

### 对话机制

- speaker 必须是主角以外的人物
- NPC 台词有个性，像真人说话——有情绪、有犹豫、有脾气
- 回应固定 2 个，方向相近，只有语气差异
- followup 不影响走向

### 决策与分支

**两种决策类型**：

**普通决策（不分支）**：
- 2-3 个选项，选后看到反馈文字，然后继续同一条线
- 反馈告诉玩家历史上真正发生了什么
- 用于丰富体验，不影响走向

**关键分支决策（决定走向）**：
- 2-3 个选项，每个选项指向不同的后续章节/场景
- 其中一个可能是死胡同（走不通，需要回退重选）
- `context` 要充分交代背景和矛盾，让玩家理解为什么这个选择很难
- 选项文字不暗示后果——玩家不知道哪条路走得通，哪条走不通
- 所有选项都应该"说得通"，死胡同不是因为选项本身荒谬，而是这条路在当时的历史情境下确实行不通

**死胡同设计**：
- 死胡同是一段 2-3 个场景的短叙事，描述这条路为什么走不通
- 叙事基于历史逻辑——不是"你死了"，而是"这条路在当时的环境下行不通，因为……"
- 死胡同末尾显示"回到上一个选择"按钮
- 死胡同叙事本身也应有趣、有信息量——玩家不会觉得浪费时间

**反馈规则（适用于普通决策）**：
- 每条 100-200 字
- 包含具体的历史事实（人名、事件、时间、引文）
- 尽量引用人物原话（用「」包裹）
- 语气温和平等——像朋友聊历史，不是老师判卷

### 章末感悟

- 每章末尾必须有 1 个 ending 场景
- 诗句优先选人物自己的作品
- annotation 必填：注明出处和白话释义
- wisdom（60-100 字）：带感情回望这段人生，让玩家被触动

### 结局系统

- 结局由分支路线决定，不是由分数决定
- 3 个结局各有独立的结局场景（叙事 + 感悟 + 诗句）
- 每个结局 150-300 字，写这条路线最终通向了什么
- 结局页展示：走过的路线回顾 + 结局叙事 + 结局诗句/名言 + 成就徽章
- 3 个结局都不批判，都有各自的意义
- "重新开始"按钮——重新开始可探索其他分支
