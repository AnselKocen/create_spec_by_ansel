### ⚠️ 核心设计理念

> **知识点、答题、动画是同一个东西的三个面，不是三个独立模块的拼接。**
>
> 每一个知识点对应一个可视化场景，这个场景贯穿「讲解→探索→答题→反馈演示」四个阶段。学生始终在同一个动画里学习和作答——讲解时动画展示现象，探索时动画开放交互，答题时动画锁定状态等待作答，反馈时动画直接演示答案。
>
> **如果把动画去掉只剩文字和选择题，这个产品就没有存在的价值。**
>
> 技术选型：
> - **SVG**：力学示意图、电路图、几何矢量图——矢量精确，支持 CSS 动画和标注
> - **Canvas + JS**：粒子运动、实时轨迹、场线流动、天体运动、流体模拟——逐帧渲染，支持大量动态元素和实时参数响应
> - **CSS 动画**：UI 过渡效果、渐变背景漂移、发光脉冲、星空粒子——提升界面酷炫感
>
> **禁止降级**：不可以用静态图片替代动画，不可以用纯文字描述替代可视化，不可以跳过探索阶段的交互功能。

## 四、视觉风格规范

> 本节强制约束游戏的视觉表现。物理/天文游戏的视觉必须达到「酷炫」级别——深空科技感，有纵深、有光影、有呼吸。

### 整体气质

**深空科技感**：学生打开页面应该感觉漂浮在太空中的高科技观测站里，四周是星云和星球，面前是精密的物理仪器面板。不是课本，是沉浸式体验。

### 背景系统

> 背景不是装饰，是氛围的基础。本项目使用全局固定背景图，所有 UI 和可视化浮在这张图之上。背景图必须由开发者在构建时编写提示词并调用生图模型生成。**禁止用 CSS 渐变、纯色或 SVG 冒充背景图。**

**背景图生成要求**：
- 提示词方向：**深空宇宙背景**——星云、星际气体云、深蓝紫色调，可散布细微星点，无主体人物，无复杂前景物体
- 参考关键词：`deep space nebula background, cosmic gas clouds, dark blue-purple tones, scattered stars, no people, no foreground objects, cinematic atmosphere, 4k wallpaper`
- 如果无法调用生图模型，**必须停下来告知用户，不可用 CSS 替代**
- **禁止用 CSS 渐变 + 白点模拟星空来冒充 AI 生成的背景图** — 深空背景看似可以用 CSS 画出来，但那不是生图，必须调用生图模型

**什么是"生图"**：
- ✅ 调用生图模型，输入提示词，获得一张真实的图像文件
- ❌ 用 CSS 渐变、box-shadow 画出来的图形 — 这不是生图
- ❌ 用 SVG path 手动描绘 — 这不是生图
- ❌ 用 Canvas 代码绘制 — 这不是生图

**实现要求**：
- `background-image` 固定铺满视口，`background-size: cover; background-position: center; background-attachment: fixed`
- 背景图上方叠加一层半透明中性灰遮罩（`rgba(0,0,0,0.40)` 左右，必须用中性灰，不可使用带色调的色值，防止遮罩影响前景文字的色温呈现）。遮罩透明度可根据页面内容密度微调——内容密集的页面（答题/讲解）遮罩可加深，内容稀疏的页面（标题页/章节过渡）遮罩可减淡
- **可选增强**：在遮罩之上叠加 1-2 个极低透明度的 radial-gradient 色块做缓慢呼吸动画（周期 ≥ 15s）

> 原则：背景图是固定的氛围底图，所有动态效果和注意力焦点在前景的可视化和 UI 上。背景不应分散对核心内容的注意力。

### 首页图层层级原则

> 背景图和遮罩必须永远位于所有 UI 内容之下，不得遮挡标题、按钮、可视化区域等任何前景元素。

- 推荐用 CSS `background-image` 属性实现背景图（作用于 body 或最外层容器），天然不会覆盖任何内容
- 如果用独立 div 实现背景图，必须确保该 div 不遮挡 UI——开发者自行选择最稳妥的实现方式
- 遮罩层同理，必须位于所有 UI 元素之下
- 粒子 Canvas 设置 `pointer-events: none`，不拦截 UI 点击事件

### 标题页氛围粒子（必须实现）

> 标题页必须有与深空主题匹配的氛围粒子效果，强化沉浸感。

- **粒子类型**：浮动星点 + 荧光粒子（小圆点，颜色取荧光蓝/荧光青/荧光紫）
- 数量 15-25 个，随机分布在背景层，缓慢漂移或闪烁
- 粒子在标题页全程可见；进入章节后可保留低密度版本作为氛围层，或关闭（不强制）
- 用 Canvas 或 CSS animation 实现均可，不影响性能
- 尊重 `prefers-reduced-motion`（关闭粒子运动，保留静态粒子）

### UI 元素风格

> 所有 UI 元素都要有「科技仪器面板」的质感——半透明、发光、精密。

**卡片与面板**：
- 半透明深色底 + `backdrop-filter: blur()` 毛玻璃效果
- 边框用渐变微光描边（`border-image: linear-gradient(...)` 或伪元素实现），不用实色边框
- 可视化区域的边框可以有缓慢流动的光效（光线沿边框移动，周期 ≥ 8s）

**按钮**：
- 透明底 + 渐变发光描边，文字用亮色
- hover：背景填充发光色 + 外发光（`box-shadow` 扩散）+ 轻微放大
- active：短暂脉冲闪烁
- 禁用态：描边和文字降至低透明度

**滑块与控件**：
- 滑轨用细线 + 微弱发光
- 滑块手柄用发光圆点，拖拽时手柄发光增强 + 轨迹高亮
- 数值显示用等宽字体，数值变化时有平滑过渡动画（不跳变）

**章节过渡**：
- 必须有仪式感：星球飞掠、粒子从中心爆散、光线扫过画面、标题文字逐字显现 + 发光
- 时长 1.5-3s，不能瞬切，也不能太慢拖沓

### SVG/Canvas 可视化风格

> 可视化是产品核心，必须精致、准确、有表现力。

**线条**：
- 精确利落，可带发光描边（SVG filter: feGaussianBlur 叠加，或多层不同透明度的 stroke）
- 线宽根据物理含义区分——力的大小映射箭头粗细和长度
- 辅助线（参考线、网格、坐标轴）用低透明度虚线，不抢主体

**填充**：
- 半透明渐变为主，用透明度表达场的强弱、能量的分布
- 天体用径向渐变模拟球体光影
- 区域填充（如力做功的面积）用极淡的半透明色块

**矢量箭头**：
- 力、速度、加速度等物理矢量用箭头表达
- 箭头长度正比于物理量大小，颜色区分不同物理量（如力用一种色，速度用另一种）
- 箭头样式统一：三角形箭头 + 圆角线帽

**标注**：
- 荧光色文字 + 细引线 + 小圆点锚点，紧贴图形
- 标注跟随运动物体时平滑移动，不跳变
- 多标注不重叠，有避让逻辑
- 数值标注用等宽字体，单位跟在数值后

**动画气质**：
- **实时模拟**——物体在力的作用下运动、轨迹实时绘出、场线实时流动、粒子实时碰撞。强调「这是活的物理」
- 参数变化时动画连续插值过渡，**禁止重绘整个画面**
- 动画帧率 ≥ 30fps，Canvas 动画使用 `requestAnimationFrame`

### 物理可视化原语速查

> 本节定义物理场景中常见图形元素的**绘制工具和质感方向**。开发者根据知识点内容自行组合，不必逐条对照——这是风格指南，不是清单。

#### 选什么工具画

| 要画的东西 | 用什么 | 为什么 |
|-----------|--------|--------|
| 矢量箭头（力/速度/加速度） | SVG `<path>` + `<polygon>` 箭头 | 矢量精确，方便标注和 CSS 动画 |
| 运动轨迹、实时模拟 | Canvas 逐帧绘制 | 需要每帧追加路径、大量运动计算 |
| 场线（电场/磁场/引力场） | Canvas 流线 | 线条多、需动态流动动画 |
| 波形（正弦波/干涉/驻波） | Canvas 逐帧重绘 | 波形每帧都在变，Canvas 更高效 |
| 光线与光路 | SVG 直线 + 滤镜辉光 | 光路是直线段，SVG 的 filter 天然适合做发光 |
| 粒子群（气体分子/电子/尘埃） | Canvas 粒子系统 | 数量多、每帧更新位置 |
| 天体（星球/轨道） | Canvas 球体 + SVG 轨道标注混用 | 球体用 radialGradient，轨道标注用 SVG 更精确 |
| 实验器材（斜面/弹簧/滑轮/电路） | SVG | 静态几何形状，需要精确尺寸和标注 |
| 数值面板、参数显示 | HTML + CSS | DOM 元素实时更新最简单 |

> **可以混用**：同一个可视化里 SVG 画器材骨架 + Canvas 画动态部分 + HTML 做数值面板，完全合理。

#### 怎么让它有质感

| 质感目标 | 一句话做法 |
|---------|-----------|
| 玻璃/透明器皿 | `rgba` 极低 alpha 填充 + 单侧白色高光条 + 半透明描边 |
| 金属/刚体 | 顶部偏亮底部偏暗的线性渐变 + 细密斜纹叠加（可选） |
| 发光/能量 | 本体下方叠一层同色模糊层（SVG `feGaussianBlur` 或 Canvas `shadowBlur`） |
| 球体/天体 | `radialGradient` 高光偏左上，边缘渐暗 |
| 液体 | 半透明鲜艳填充 + 弯月面弧线顶部 + 可选气泡小圆 |
| 场/力等不可见的东西 | 用线条密度表强弱 + 沿线流动的小光点表方向 + 辉光表能量 |

#### 颜色怎么分配

> 不指定具体色值，指定**分配规则**——保证同一画面里不同物理量一眼可辨。

- 不同物理量（力 vs 速度 vs 加速度）**必须用不同色系**
- 同一物理量的分量用同色系的降饱和或虚线版本
- 光线用自身频率对应的颜色（白光=白/浅黄，红光=红，紫光=紫）
- 温度/能量用冷暖色谱（蓝=冷/低 → 红=热/高）
- 正确 vs 错误对比用成功色 vs 错误色（已在配色方案中定义）

#### 标注原则

- 关键物理量旁边**紧贴标注**（符号 + 数值 + 单位），用等宽字体
- 引线从标注连到目标元素，末端一个小锚点圆
- 数值变化时**平滑过渡**，不跳变
- 多标注不重叠，静态手动错开，动态加简单避让
- 答题阶段可**选择性隐藏**部分标注作为题目条件

#### 交互态视觉暗示

- 可拖拽的元素：加微弱呼吸辉光 + hover 放大 + cursor: grab
- 答题锁定态：辉光消失 + 降 opacity + cursor: default
- 拖拽中：辉光最亮 + 微微放大
- 释放：快速回弹

### 配色方案

| 用途 | 方向 | 参考值（不强制） | 说明 |
|------|------|-----------------|------|
| 60% 主色（背景/表面） | 深空色系（与背景图一致） | 如 #0a0e1a / #1a1030 | 深蓝黑 / 深紫黑，面板底色需与背景图的蓝紫色调融合 |
| 30% 辅助色（文字/边框） | 冷亮色 | 如 #e2e8f0 | 浅蓝白 / 银灰 / 冷白，保证可读性 |
| 10% 强调色（按钮/高亮） | 荧光色 | 如 #00d4ff | 荧光蓝 / 荧光青 / 电光紫，用于交互元素 |
| 成功色 | 亮绿系 | 如 #00ff88 | 答对反馈、正确路径高亮 |
| 错误色 | 暖红系 | 如 #ff4757 | 答错反馈，不刺眼 |
| 可视化色板 | 多色区分 | — | 不同物理量用不同色，饱和度适中，互相有辨识度 |

> 配色原则：遵循 60-30-10 法则。可视化元素使用高饱和度色彩突出关键信息，文字与背景对比度 ≥ 4.5:1。具体色值由开发者在以上方向内自行决定。

| 用途 | 字体 | 字号 | 字重 |

- **间距基准**：8px 网格（8 / 16 / 24 / 32 / 40）
- **卡片内边距**：16-24px
- **模块间距**：16-24px

### 必须实现的视觉反馈

| 用户操作 | 视觉反馈要求 |
|---------|------------|
| 进入章节 | 章节过渡动画（星球飞掠/粒子爆散 + 标题显现），不能瞬切 |
| 讲解阶段 | 文字逐段显现，可视化动画同步播放 |
| 探索阶段 | 拖拽/调参时可视化实时响应 + 数值面板平滑更新 |
| 选择答案 | 选项 hover 发光高亮，选中后锁定 |
| 答对 | 可视化播放正确结果 + 成功光效 + 鼓励文字 |
| 答错 | 可视化演示正确过程 + 温和提示 + 可重试 |
| 解锁成就 | toast 通知从顶部滑入，成就图标 + 名称 + 描述，自动消失 |
| 完成章节 | 知识卡片总结 + 成就展示 |
| 屏幕切换 | 淡入淡出过渡，不能瞬切 |
| 悬停可交互元素 | 发光/放大/颜色变化，必须有明确反馈 |

### 动效标准

- **标题页进场**：背景图 → 主标题上浮淡入 → 副标题淡入 → 按钮上浮淡入，依次触发，总时长 ≤ 1.5s
- **标题页按钮**：常态有发光脉动（box-shadow 呼吸，周期 3s），hover 发光扩散 + 轻微放大，click 短暂脉冲压缩
- **标题页 → 章节主页**：星光粒子向中心汇聚或光线扫过，内容淡出淡入，不瞬切（800ms 左右）
- 所有 UI 过渡时长 200-800ms
- 使用 ease 或 cubic-bezier 缓动曲线，禁止线性运动
- 背景动画周期 ≥ 15s，极缓慢
- 尊重用户的 `prefers-reduced-motion` 系统偏好（关闭背景粒子和光晕动画，保留可视化核心动画）
- 可视化动画应能暂停/重播
- 动效必须有目的（辅助理解 / 提供反馈 / 营造氛围），禁止无意义的闪烁

### 核心路径

```
标题页 → 章节主页（显示所有章节解锁状态）→ 选择当前可进入的章节 → 章节过渡 → [知识点闭环: 讲解(动画) → 探索(交互) → 答题(选择) → 反馈演示(动画)] × 3-4个知识点 → 章末总结 + 成就 → 返回章节主页（下一章解锁）→ ... → 结局总结(成就墙)
```

## 七、信息结构

### 导航结构

```
标题页
├── 开始学习 → 章节主页
└── 继续学习 → 章节主页（恢复存档进度，仅有存档时显示）

章节主页（核心导航枢纽，每次完成章节后返回这里）
├── 章节列表（纵向或横向排列）
│   ├── 已完成章节 → 亮色 + 完成标记 + 可点击回看
│   ├── 当前可进入章节 → 高亮 + 引导动效 + 可点击进入
│   └── 未解锁章节 → 灰色/锁定 + 不可点击
└── 整体进度展示（如 2/6 章已完成）

学习中（线性推进，章内不可跳转）
├── 章节过渡屏
├── 知识点闭环（重复 3-4 次/章）
│   ├── 讲解阶段（动画 + 文字）
│   ├── 探索阶段（交互式可视化）
│   ├── 答题阶段（3 选 1）
│   └── 反馈演示（动画演示答案）
└── 章末总结（知识卡片 + 成就展示）→ 返回章节主页

结局总结页（全部章节完成后触发）
├── 成就墙（所有已解锁成就）
├── 各章回顾
└── 返回章节主页
```

> 本游戏为线性解锁路径——必须通过第 N 章才能解锁第 N+1 章。但章节主页允许回看已完成章节。

### 游戏界面布局

| 界面 | 核心内容 | 交互方式 |
|------|---------|---------|
| 标题页 | 游戏名称（主题化字体特效）+ 副标题 + AI 生成背景图（遮罩减淡）+ 开始/继续按钮 + 氛围粒子 | 点击按钮 |
| 章节主页 | 章节列表（已完成/当前/未解锁状态区分）+ 整体进度 | 点击章节进入 |
| 章节过渡 | 章标题 + 本章核心概念预告 + 过渡动画 | 点击"进入" |
| 讲解阶段 | 上方：SVG/Canvas 可视化区域（动画播放）；下方：讲解文字（逐段显现）+ 继续按钮 | 点击继续 |
| 探索阶段 | 中央：交互式可视化（带拖拽/滑块）；下方：参数面板 + 引导文字 + "来挑战"按钮 | 拖拽/滑动 + 点击 |
| 答题阶段 | 上方：可视化（锁定特定状态）；下方：题目文字 + 3 个选项按钮 | 点击选项 |
| 反馈演示 | 上方：可视化（播放答案演示）；下方：反馈文字 + 继续/重试按钮 | 点击继续或重试 |
| 章末总结 | 公式卡片 + 要点列表 + 本章成就展示 | 点击继续 |
| 结局总结 | 成就墙（卡片网格）+ 各章一句话回顾 + 重新开始按钮 | 点击"重新开始" |

### 标题页设计规范

**字体层级**：
- 主标题字号 ≥ 40px（移动端）/ ≥ 56px（桌面端），有深空风格字体特效（干净有力的字体 + 发光投影或亮色填充）。标题基色必须可见，特效是增强而非前提
- 副标题字号约为主标题的 40-50%，字重轻，颜色比主标题暗淡，体现层次
- 按钮文字字间距略宽，有科技感

**进场动画**（页面加载时依次触发）：
1. 背景图淡入（300ms）
2. 粒子开始漂浮
3. 主标题上浮淡入（500ms，delay 200ms）
4. 副标题淡入（400ms，delay 600ms）
5. 按钮上浮淡入（400ms，delay 900ms）

**按钮动效**：
- 常态：发光描边呼吸脉动（box-shadow 周期 3s）
- hover：外发光扩散 + 轻微放大（scale 1.05）
- click：短暂脉冲压缩（scale 0.97）

**标题页 → 章节主页过渡**：点击按钮后，标题页内容淡出，可加星光汇聚或光线扫屏效果（800ms），背景图保持不动

## 九、交互设计

### 交互模式选择

**本项目选择**：新手友好 + 探索式

**选择原因**：目标用户是学生，基本操作（点击推进 + 选择答案）要极简，但探索阶段需要拖拽/滑块等交互来建立物理直觉。

### 操作映射

| 操作 | 实现方案 | 说明 |
|------|---------|------|
| 推进讲解 | 点击"继续"按钮 | 文字和动画同步，按钮在动画播完后出现 |
| 交互探索 | 拖拽物体/箭头/滑块 | 可视化实时响应，数值面板同步更新 |
| 回答问题 | 点击 3 个选项之一 | 选后进入反馈演示，答错可重试 |
| 观看反馈 | 自动播放 + 点击继续 | 反馈动画播完后出现继续按钮 |
| 存档 | 自动 | 每次状态变化自动写入 localStorage |
| 读档 | 点击标题页"继续学习" | 进入章节主页，恢复存档进度 |

### 交互探索的具体模式

> 探索阶段的交互方式取决于物理内容，以下为常见模式：

| 交互模式 | 适用场景 | 操作方式 |
|---------|---------|---------|
| 拖拽矢量 | 力的方向和大小 | 拖拽箭头尾部改方向，拖拽头部改大小 |
| 拖拽物体 | 改变初始位置/条件 | 直接拖拽物体到新位置 |
| 滑块调参 | 连续变量（角度、速度、质量、电荷…） | 拖拽滑块，数值面板实时更新 |
| 开关切换 | 显示/隐藏辅助线、力的分解、参考系切换 | 点击 toggle 按钮 |
| 播放/暂停 | 运动模拟过程 | 点击播放按钮，可暂停在任意时刻观察 |

### 交互反馈

| 操作 | 视觉反馈 | 状态变化 |
|------|---------|---------|
| 阅读讲解 | 文字显现 + 动画播放 | 进入探索阶段 |
| 拖拽参数 | 可视化实时变化 + 数值更新 | 无状态变化 |
| 答对 | 可视化演示正确结果 + 成功光效 + 鼓励文字 | 可能触发成就 |
| 答错 | 可视化演示正确过程 + 温和提示 + 重试按钮 | 记录错误，不扣分 |
| 探索中特殊操作 | 可能触发创意成就的 toast | 解锁隐藏成就 |
| 进入新章 | 章节过渡动画 | currentChapter +1 |
| 通关 | 跳转结局成就墙 | finished = true |

## 十、核心机制约束

> 以下是**不可更改的规则**，开发时必须遵守。

### 科学准确性（极其重要）

- 公式、数值、单位必须正确——**这是本产品的底线**
- 可视化必须真实反映物理定律，不能为了好看而牺牲准确性
- 探索阶段的参数范围必须物理上合理（如角度 0-360°，质量 > 0，速度不超过 c）
- 解析必须严谨，不能有模糊或误导性表述
- 题目的正确答案必须经过验证

### 知识自检规则

> **AI 在生成内容时，必须对所有涉及的物理知识执行自检。** 这是强制流程，不可跳过。

自检清单（每个章节输出后逐条核对）：
1. **公式验证**：本章出现的所有公式是否正确？单位是否匹配？
2. **数值验证**：题目中的计算过程是否正确？答案是否经过反算验证？
3. **概念验证**：讲解中的物理概念是否准确？有没有常见的误解被当作正确知识？
4. **可视化一致性**：可视化中的物理行为是否与公式一致？（如 r = mv/qB，速度增大时半径确实增大）
5. **选项设计**：错误选项是否对应真实的学生常见错误（不是随机干扰项）？正确选项的解析是否充分？

> 如果自检发现错误，必须修正后再输出。在输出末尾附上自检结果摘要。

### 知识点四阶段闭环规则

> **这是本游戏区别于普通题库的核心机制。** 每个知识点必须走完四个阶段，且四个阶段共用同一个可视化。

**阶段 1：讲解**
- 一边说原理一边播动画，学生看到物理现象
- 可以分多段讲解（1-3 段），每段 60-120 字，每段对应不同的 vis_state（如：先讲现象→再讲原因→最后引出公式）
- 通俗易懂，先给直觉再给公式
- 关键术语首次出现时加粗并解释
- 动画和文字同步——文字说到力，动画就展示力

**阶段 2：探索**
- 开放交互参数，学生自己拖拽/调节
- 提供引导文字（如"试试把角度调到 90°，看看力的变化"）
- 探索没有对错，纯粹建立直觉
- 可视化实时响应参数变化，数值面板同步更新
- 某些特殊操作（如把参数拉到极端值）可触发创意成就

**阶段 3：答题**
- **题目必须基于前两个阶段的可视化出题**——考察学生对可视化现象的理解
- 可视化锁定在特定状态（如锁定某参数、隐藏某数值）
- 恰好 3 个选项，正确答案只有 1 个
- 选项文字清晰无歧义

**阶段 4：反馈演示**
- **不管答对还是答错，都用可视化演示**
- 答对：可视化播放正确结果 + 高亮关键变化 + 肯定 + 为什么对（30-80 字）
- 答错：可视化先演示「你选的会怎样」再演示「正确的是怎样」+ 不批评 + 指出错在哪（50-100 字）
- 答错可以重试（但不重复触发成就）
- 语气鼓励——"没关系，我们看看实际会怎样" 而不是 "错了"

### 四阶段的可视化一致性

- **禁止**每个阶段造一个不相关的可视化——四个阶段是同一个可视化的不同状态
- 讲解时：动画处于「演示模式」，按预设流程播放
- 探索时：动画处于「交互模式」，参数开放
- 答题时：动画处于「锁定模式」，特定参数固定
- 反馈时：动画处于「演示模式」，播放答案对应的过程

### 成就系统规则

> **本游戏不使用分数制。** 用成就替代评分，每章 3 个成就，成就要有个性和记忆点。

**每章 3 个成就**：

| 类型 | 触发条件 | 风格要求 |
|------|---------|---------|
| 全对成就 | 本章所有题首次答对 | 正面肯定，专家/大师风格。如"电磁学大师"、"轨道力学专家" |
| 全错成就 | 本章所有题首次答错 | 调皮调侃，不伤人。如"对万有引力真是一点儿也不了解~"、"电荷们在为你哭泣" |
| 创意成就 | 探索阶段的特殊操作 | 彩蛋风格，奖励好奇心。如"把速度拉到光速的人"、"发现了拉格朗日点" |

成就规则：
- 成就名称 6 字以内，描述 30-50 字
- 解锁时弹出 toast 通知（从顶部滑入，3s 后自动消失）
- 结局页展示成就墙：已解锁的成就亮色显示，未解锁的灰色占位（可显示提示）
- 重试不重复触发成就
- 全对/全错判定基于**每道题的首次作答**——答错后重试答对不影响"首次答错"的记录
- 触发时机：章内**最后一道题首次作答完成后**立即判断，若条件满足则弹出 toast；不在每道题后单独判断
- 创意成就的触发条件由开发者根据可视化内容设计，鼓励学生大胆探索
- "重新开始"会清除所有进度和成就，从头来过

### 章末知识总结

- 每章末尾 1 个总结场景
- 核心公式大字居中展示；本章有多条核心公式时逐条展示，数据来自 `summary.formulas` 数组
- 2-3 个要点（bullet points）
- 一句话点睛（让学生记住的核心直觉）
- 展示本章已解锁的成就

### 结局总结

- 成就墙：所有成就卡片网格排列，已解锁的亮色 + 解锁动画，未解锁的灰色轮廓
- 各章一句话回顾（核心知识点提炼）——数据来自各章 `summary.insight` 字段，直接复用，不另行生成
- 不展示分数、百分比、评级
- "重新开始"按钮

> **核心设计：一个知识点 = 一个可视化。** `visualization` 在 knowledge_point 级别定义一次，所有阶段通过 `vis_state` 描述同一可视化在不同阶段的状态。一个知识点通常 1 道题，复杂知识点可出 2 道（基础+进阶），但共用同一个可视化。禁止每个阶段各自造不相关的可视化。

### 物理场景速查

> 以下列出常见物理可视化场景的要点，帮助开发者快速定位。

**力学**：
- 力的箭头分解与合成——箭头首尾相接，合力用不同颜色
- 抛体运动——轨迹实时绘出，速度/加速度箭头跟随
- 圆周运动——向心力始终指向圆心，速度沿切线
- 弹簧振子——简谐运动 + 能量条（动能/势能此消彼长）
- ⚠️ 常见错误：力的方向画反、合力箭头不从作用点出发

**电磁**：
- 磁场线——从 N 到 S，疏密反映场强，可拖拽磁极
- 安培力——电流方向 × 磁场方向 = 力方向，左手定则
- 带电粒子在磁场中运动——圆形轨迹，半径 r = mv/qB
- ⚠️ 常见错误：场线交叉、洛伦兹力方向与速度方向平行

**波动/光学**：
- 波形传播——正弦波逐帧推进，可调频率/振幅/波速
- 干涉/叠加——两列波相遇时的合成波形实时计算
- 光路折射反射——入射角/折射角/法线标注清晰
- ⚠️ 常见错误：折射时角度不满足 Snell 定律

**天文**：
- 天体轨道——椭圆轨道 + 焦点标注，可调偏心率
- 万有引力——两天体间的引力箭头，大小随距离变化
- 开普勒定律——等面积速率的扫过区域动画
- ⚠️ 常见错误：轨道画成正圆、引力与距离成正比（应该是反比平方）

**热学/粒子**：
- 粒子随机运动——Canvas 大量粒子，温度 → 粒子速度
- 气体压强——粒子撞击容器壁的动画
- ⚠️ 常见错误：粒子运动方向过于规律（应该随机）
